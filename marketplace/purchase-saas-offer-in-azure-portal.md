# <a name="purchase-a-saas-offer-in-azure-portal"></a>Azure portal で SaaS プランを購入する

この記事では、Azure portal からサービスとしてのソフトウェア (SaaS) プランを検索、試用、購入するためのさまざまなオプションと要件について説明します。

## <a name="saas-offers-discovery-in-azure-portal"></a>SaaS は Azure portal での検出を提供します

Azure portal したら、SaaS プランに焦点を絞るために、いくつかの方法で検索を絞り込むことができます。

### <a name="narrowing-your-search"></a>検索範囲を絞り込む

ホームページで、[ **Azure サービス** ] の下にある [ **+ リソースの作成** ] または [ **Marketplace**] を選択します。 または、プラットフォームの任意の場所でショートカット **G + N** を使用します。

- [ **オファーの種類** ] フィルターを使用して saas プランに結果を絞り込んだ後、[ **saas**] を選択します。
- 特定の SaaS プランを検索するには、上部のナビゲーション領域にあるグローバル検索バーを使用します。

**Marketplace** のホームページの上部にあるバナーを選択して、[プライベート SaaS プラン](/marketplace/private-offers)を検索します。 すべてのプランまたはプランがすべての地域で使用できるわけではありません。また、一部のテナントに対してのみ表示される場合もあります。

フィルター処理されたビューには、タイトルで表される使用可能な各 SaaS プランが表示されます。1つを選択すると、製品の詳細ページが表示されます。 これには、次のセクションが含まれます。

- 概要–サービス、マーケティング、および学習資料の詳細
- プラン + 価格–各プランには、請求条件と価格が異なるプランが少なくとも1つ含まれます
- 使用情報 + サポート–発行者 ID、プラン ID、およびプラン ID を含みます。
- 特定の SaaS プランの評価とレビュー

## <a name="available-billing-models-plansskus-for-saas-offers"></a>SaaS プランの利用可能な課金モデル (プラン/Sku)

各 SaaS プランには1つ以上のプランがあります。 各プランには、定額料金またはユーザーごとの料金モデルが関連付けられています。 各プランの価格は、月額料金であり、ゼロドルになる可能性があります (表示されている価格は、実際のコストを反映するためのものではありません)。 この料金は、定額料金またはユーザーごとの料金です。 使用可能なプランの種類:

- **月単位プラン** –定期的な月額料金。月単位の定期的な月額料金またはユーザー単位の月額料金。 期間が終了すると、プランは自動的に更新されます。
- **年間プラン** –定期的な年間料金。年間の繰り返しに対して支払われる、定額またはユーザー単位の年間料金。 期間が終了すると、プランは自動的に更新されます。
- **カスタムメーター** –定期的な料金と共に、定額割引プランには、定額法に含まれていない超過使用量に対して、オプションのカスタム測定ディメンションを含めることもできます。 各ディメンションは、課金対象の単位を表します。 これは、帯域幅、チケット、処理された電子メールなど、従量制課金単位の使用量に応じて変化する変動コストです。 これらのディメンションの使用量に応じて月単位で課金されます。 超過分の消費は、定額率に含まれるすべての従量制課金ユニットを使用した場合にのみ開始されることに注意してください。
- **無料試用版** -プランには、1か月の試用期間が含まれており、その間はソフトウェアを無料で使用できます。  試用期間が終了すると、プランに従って課金されます。 試用版プランはカスタムメーターと互換性がありません。

これらの価格モデルは、パブリックとプライベートの両方のプランで使用できます。

## <a name="saas-purchase-experience"></a>SaaS の購入エクスペリエンス

1. [製品] ページで、ニーズに合ったプランを選択し、セットアップ **を続行** します。
2. 購入プロセスの一環として、[ **基本** ] タブにリダイレクトされるので、次の操作を行う必要があります。
    1. 課金に使用する *サブスクリプション* を定義します。 使用する Azure サブスクリプションには、有効な購入方法が定義されている必要があります。 適切なレベルのアクセス許可を持っているか、適切なレベルのアクセス許可を持つサブスクリプションの下にリソースグループを用意する必要があります。 また、請求先の国は、プランを購入できる国である必要があります。 有効な支払い方法を使用していない Azure サブスクリプション (MSDN サブスクリプションなど) は、無料プランの購入にのみ使用できます
    1. SaaS リソースが属する *リソースグループ* を選択または作成します。
    1. 後で簡単に識別できるように、SaaS サブスクリプションの *名前* を入力します。 購入した後は、名前を変更することはできません。
    1. [ **プラン**] で、製品の詳細ページ (PDP) で選択したプランが表示されます。 PDP でアクティブな選択を行っていない場合は、既定のプランが表示されます。 [ **プランの変更** ] リンクを選択すると、選択内容を変更できます。 関連する請求期間を選択し、別のプランを選択します。 購入後にプランを変更できる場合があります (パブリッシャーでサポートされている場合)。 ただし、この期間を月単位から年単位、または年間から月単位に変更することはできません。
    1. 料金モデルが *ユーザー単位* の場合は、 *ユーザー* 数の指定が必要になることがあります。 表示される料金は、選択したサブスクリプション、プラン、および用語に応じて変わります。
3. [ **タグ** ] タブに進みます。 *タグ* は、ユーザー定義のキーと値のペアであり、リソースまたはリソースグループに直接配置できます。 タグを使用すると、後で SaaS リソースを簡単に見つけることができます。 現在、Azure では、1 つのリソースまたはリソース グループにつき最大 50 個のタグがサポートされます。 タグは、リソースの作成時に付けたり、既存のリソースに追加したりすることができます。
4. 「 **レビュー + サブスクライブ** 」に進み、プランとプランの詳細を確認します。
    1. 発行元および Azure Marketplace の *利用規約*、修正、*プライバシーポリシー* *を確認* する
    1. 連絡先の詳細を追加するように求められる場合があります
    1. *基本* と *タグ* の詳細の確認
5. 確認ができたら、[ **サブスクライブ**] を選択します。

## <a name="saas-subscription-and-configuration"></a>SaaS サブスクリプションと構成

[Subscribe] を選択すると、"SaaS サブスクリプションは進行中です" というメッセージが表示されます。 この処理には数分かかることがあります。完了するまでウィンドウを閉じないでください。

サブスクリプションが完了すると、SaaS サブスクリプションが完了したことを示すメッセージが表示され、購入を楽しんで開始するようにアカウントを構成する必要があります。 また、新しいサブスクリプションのアクティブ化を要求する電子メールも受信します。 SaaS アカウントを構成するユーザーでない場合は、このメールを関連するユーザーに転送します。

このプロセスを完了して SaaS の使用を開始するには、サブスクリプションの構成を開始する必要があります。 [ **今すぐアカウントを構成** ] ボタンを選択すると、発行元の web サイトにリダイレクトされます。

ヘッダーの右上隅にある [ベル] アイコンを選択して、サブスクリプションの状態を確認することもできます。

*30 日* 以内に構成プロセスを完了しない場合、この SaaS サブスクリプションは自動的に *削除* されます。 課金は、発行元の web サイトでアカウントを構成した後に開始されます。
