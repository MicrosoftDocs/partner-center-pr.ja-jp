## <a name="price-changes"></a>価格の変更

価格の変更は一般的に発生します。 パートナーは、価格表のプレビューを見て、ライセンスベースのオファーの価格の変化を予測できます。 ダッシュボードパートナー センター、[価格とプラン] ページを開き、価格表のプレビューを表示します。

ただし、Azure の使用量ベースの価格にはプレビューはありません。 パートナーは、RateCard API を使用して Azure の消費価格の変更に対応できます。この API は、その日の測定価格を返します。

|**製品の種類**   |**製品の例**  |**利用可能なプレビュー** |**変更の詳細**|
|-----------------------|:-----------------------|:-------------------|------------------|
|ライセンスベース|Office、Dynamics、Intune、Windows Enterprise|30 日間|プレビュー価格表で CHNG とマークされた価格の変更を表示する|
|使用量ベース|Azure リソース|使用不可|前月の価格表の [変更履歴] タブで使用できる **変更** ログ|
|ソフトウェア||使用不可|月から月の価格表を手動で比較する|
|Reservations|仮想マシン、前払い|使用不可|月から月の価格表を手動で比較する|

使用量ベースの価格は、1 か月を通して変化する可能性があります。 これらの Azure リソースの "最新" の毎日の価格を取得するには、パートナーが RateCard API を呼び出す必要があります。 

> [!Note] 
> サブスクリプションの価格の変更は、更新中にのみ適用されます。 パートナーの月額料金は、購入価格、またはサブスクリプション作成時の価格で決定されます。 年間の期間を取得した後に価格が増減した場合、パートナーは更新されるまで変更された価格 (通常は 12 か月の期間) に課金されません。

## <a name="pricing-between-azure-and-non-azure"></a>Azure と Azure 以外の価格

価格は、オファーの種類によって異なります。 通常、ライセンスベースの価格は、特定の月のライセンスあたりの金額です。 使用量ベースの価格は、関連付けられたメーター ID を持つ特定のリソースの使用によって決定されます。 パートナーは、Azure サブスクリプションの取得に対して課金されません。 ただし、パートナーは、Azure サブスクリプションの異なるデプロイによって消費されるリソースに対して課金されます。 使用量ベースの価格表の価格は、Azure の異なるリソース メーターの ID を中心に編成されています。

Azure の予約は、特定のリソースの種類に対する用語ベースの購入Virtual Machines。 Azure の予約を購入すると、パートナーは前払い (1 年または 3 年の期間) を行い、特定の仮想マシンを予約できます。 予約によってパートナーの料金が節約され、その仮想マシンが期間の間常に使用可能になります。 パートナーは、必要な予約を使用量ベースのリソースメーターの ID と一致できます。 メーターの ID は、パートナーが仮想マシンを購入しているか、単に仮想マシンを使用量ベースのリソースとしてデプロイしているかに関して、リソース全体で一貫しています。

## <a name="offer-limits"></a>オファーの制限

一部のライセンスベースのオファーには、同じ顧客に対する複数の購入を禁止する特定のルールと制限があります。 これらのルールは、ほとんどの試用版と多くの小規模ビジネス オファーに適用されます。 **小規模ビジネス オファーは** 、最大ライセンス数が 300 未満のオファーによって定義されます。

これらの購入の制約は、オファー構成の一部として定義され、オファー一覧マトリックスを参照して確認できます。 2 つのデータ列が組み合わせて適用を定義します。1。 オファーの制限スコープと 2。 オファーの制限。 制約は購入時に適用されます。 パートナー センターのカタログでは、パートナーがルールで許可されている数を超えるオファーを購入する許可を得てはくれ" と表示されます。 制約に違反しようとすると、エラーが発生します。

オファーの制限スコープは、オファーリスト マトリックスの列として記録され、None、Lifetime、または Concurrent の値を持つ場合があります。 

- [なし **] のオファー** は、制約なしで購入できます。
- **有効期間** オファーは 1 回だけ購入できます。
- **同時** オファーは、そのオファーの [オファーの制限] の値で許容される回数まで購入できます。 ほとんどの試用版には、"1" のオファー制限を持つ有効期間オファー制限スコープがあります。 ほとんどの小規模なビジネス オファーには、オファーの上限が "2" の同時オファー制限スコープがあります。

> [!IMPORTANT]
> オファーが取り消された場合でも、コンカレンシーの制限が適用されます。 追加の領域を解放して別の購入を許可するには、オファーを完全に取り消してからプロビジョニング解除する必要があります。

## <a name="multi-year-term-offers"></a>複数年の期間のオファー

### <a name="36-month-offers"></a>36 か月のオファー

約 50 の Dynamics オファーには、3 年間の期間があります。 これらは、オファーの **タイトルの (36 mo)** で識別されます。 これらのオファーは、毎年の用語オファーに似ています。 唯一の違いは、用語に関する点です。 これらのオファーには 3 年間の期間があります。そのため、サブスクリプションは 1 年ではなく 3 年後に自動的に更新されます。 これらのオファーの動作の概要を次に示します。

- 使用条件は 36 か月で、サブスクリプションは 3 年後に自動的に更新されます
- パートナーはサブスクリプションの期間中、ライセンスの数を取り消したり、変更したりできます。
- 年間更新は、3年間の購入時間の料金になります。
- 請求頻度は、毎年または毎月です
