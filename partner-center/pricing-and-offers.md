## <a name="price-changes"></a>価格の変更

価格の変更は一般的に発生します。 パートナーは、価格表のプレビューを見て、ライセンスベースのオファーの価格の変化を予測できます。 ダッシュボードパートナー センター、[価格とプラン] ページを開き、価格表のプレビューを表示します。

ただし、Azure の使用量ベースの価格にはプレビューはありません。 パートナーは、RateCard API を使用して Azure の消費価格の変更に対応できます。この API は、その日の測定価格を返します。

|**製品の種類**   |**製品の例**  |**利用可能なプレビュー** |**変更の詳細**|
|-----------------------|:-----------------------|:-------------------|------------------|
|ライセンスベース|Office、Dynamics、Intune、Windows Enterprise|30 日間|プレビュー価格表で CHNG とマークされた価格の変更を表示する|
|使用量ベース|Azure リソース|使用できません。|前月の価格表の [変更履歴] タブで使用できる **変更** ログ|
|ソフトウェア||使用できません。|月から月の価格表を手動で比較する|
|Reservations|仮想マシン、前払い|使用できません。|月から月の価格表を手動で比較する|

使用量ベースの価格は、1か月を通じて変更できます。 これらの Azure リソースに対して "現在" の日単位の価格を取得するには、パートナーは RateCard API を呼び出す必要があります。 

> [!Note] 
> サブスクリプション価格の変更は、更新中にのみ適用されます。 パートナーの月額料金は、購入価格またはサブスクリプションを作成したときの料金によって決まります。 年間契約期間を取得した後に価格が増減した場合、パートナーは更新されるまで変更価格を請求されません。通常は12か月の期間です。

|**Segment**   |**認定を受ける必要があるユーザー**   |**パートナーの顧客の見極め**|**有効な製品の種類**|

## <a name="add-on-offer-types"></a>アドオン オファーの種類

ライセンス ベースのサービスは、基本オファーまたはアドオンとして取得できます。 基本オファーのみを検出し、カタログから購入パートナー センターできます。 基本オファーを購入した後、アドオンを適用する必要があります。 ライセンス ベースの価格表の [セカンダリ ライセンスの **種類] 列には** 、各オファーとその種類に関する情報が含まれています。 基本オファーの **価格表のセカンダリ** ライセンスの種類列には非固有の値が含まれています。カタログで購入できます。 アドオンのセカンダリ ライセンス **の種類の値** は、カタログで購入できません。 これらのアドオンを購入するには:

1. アドオンを購入する前に購入する必要があるオファーの一覧を確認するには、オファー一覧マトリックスを参照してください。
2. カタログから基本オファーを購入する
3. 顧客一覧から顧客に移動します。 購入したベース オファーのサブスクリプションをクリックします。 [サブスクリプションの管理] ページに、基本プランに適用できる使用可能なアドオンが表示されます。

> [!Note] 
> 一部の基本オファーには **、アドオン** ライセンスの単位 **の種類の値があります**。 基本プランの場合、これは購入後にユーザーライセンスを割り当てないことを意味します。 プランをカタログで購入できる場合は、ユーザーインターフェイスの単位の種類に関係なく **基本プラン** になります。

## <a name="pricing-between-azure-and-non-azure"></a>Azure と Azure 以外の価格

価格は、プランの種類によって異なります。 ライセンスベースの価格は、通常、特定の月のライセンスあたりの金額です。 使用量ベースの価格は、特定のリソースと、関連付けられているメーター ID を使用して決定されます。 パートナーは、Azure サブスクリプションの取得に対して課金されることはありません。 ただし、パートナーは、Azure サブスクリプションのさまざまなデプロイで消費されるリソースに対して課金されます。 使用量ベースの価格表の価格は、Azure のさまざまなリソースメーター Id に基づいて構成されています。

Azure の予約は、特定のリソースの種類 (Virtual Machines) に対して、用語ベースで購入します。 Azure 予約を購入すると、パートナーは事前に (1 年または3年)、特定の仮想マシンを予約できます。 予約は、パートナーの費用を節約し、その期間中は常に仮想マシンを使用できるようにします。 パートナーは、使用量ベースのリソースメーター Id に対して、必要な予約を揃えることができます。 測定 Id は、パートナーが仮想マシンを購入しているか、単に使用量ベースのリソースとして仮想マシンをデプロイしているかにかかわらず、リソース全体で一貫しています。

## <a name="offer-limits"></a>プランの制限

一部のライセンスベースのプランには、同じ顧客に対して複数の購入を禁止する特定のルールと制限があります。 これらの規則は、ほとんどの試用版と小規模なビジネスプランに適用されます。 **小規模ビジネス オファーは** 、最大ライセンス数が 300 未満のオファーによって定義されます。

これらの購入の制約は、オファー構成の一部として定義され、オファー一覧マトリックスを参照して確認できます。 2 つのデータ列が組み合わせて適用を定義します。1。 オファーの制限スコープと 2。 オファーの制限。 制約は購入時に適用されます。 パートナー センターのカタログでは、パートナーがルールで許可されている数を超えるオファーを購入する許可を得てはくれ" と表示されます。 制約に違反しようとすると、エラーが発生します。

オファーの制限スコープは、オファーリスト マトリックスの列として記録され、None、Lifetime、または Concurrent の値を持つ場合があります。 

- [なし **] のオファー** は、制約なしで購入できます。
- **有効期間** オファーは 1 回だけ購入できます。
- **同時** オファーは、そのオファーの [オファーの制限] の値で許容される回数まで購入できます。 ほとんどの試用版には、"1" のオファー制限を持つ有効期間オファー制限スコープがあります。 ほとんどの小規模なビジネス オファーには、オファーの上限が "2" の同時オファー制限スコープがあります。

> [!IMPORTANT]
> オファーが取り消された場合でも、コンカレンシーの制限が適用されます。 追加の領域を解放して別の購入を許可するには、オファーを完全に取り消してからプロビジョニング解除する必要があります。
