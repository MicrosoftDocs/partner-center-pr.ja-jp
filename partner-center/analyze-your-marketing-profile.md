# <a name="analyze-your-leads---see-how-well-you-attract-target-customers-and-potential-referrals"></a>潜在顧客を分析する - ターゲット顧客と見込みのある紹介を把握していることを確認する
<!-- 
https://go.microsoft.com/fwlink/?linkid=849120
-->

**適切なロール**: 紹介管理者

[ **紹介]** の [ **分析] セクションの** [リード] ページでは、紹介がどのように実行されているのか確認できます。 これらのメトリックを定期的に確認し、注意が必要な傾向や分野を特定し、ビジネス目標に向けて取り組んでください。

リード分析情報データを表示するには、パートナー センター メニューから[紹介] に移動し、[>分析] > **します**。

## <a name="apply-filters"></a>フィルターを適用する

[リード] ページ **の上部** 付近で、データを表示する期間を選択できます。 既定の選択は **[3M]** (3 か月) ですが、6 か月または 1 年の期間のデータを表示することもできます。 また、 **[カスタム]** を選択して、特定の期間のすべての紹介を表示することもできます。

[フィルター] ボタンをクリックすると、パネルが開きます。このパネルでは、顧客名、国、案件の種類、案件の方向、ソリューション名、および状態に基づいて、このページのすべてのデータをフィルター処理できます。 以下にこれらのフィルターの詳細を示します。

- **顧客名**:既定値は **[すべて]** ですが、選択した 1 人または複数の顧客にデータを絞り込むことができます。
- **国**:既定値は **[すべて]** ですが、選択した顧客の 1 つまたは複数の国にデータを絞り込むことができます。
- **紹介プログラム**: 既定値は **[すべて**] ですが、データを特定の紹介プログラムに制限できます。 このフィルターは、組織が紹介プログラムに登録されている場合にのみ表示されます。
- **状態**: 既定値は **[すべて]** ですが、選択した 1 つまたは複数の紹介の状態の種類 (承諾、辞退、期限切れ、失注、受注など) を含む紹介にデータを絞り込むことができます。
- **認定の** 種類: 既定値は **[すべて**] ですが、マーケットプレースのリードまたは有資格リードである紹介にデータを制限することができます。

ここで説明するすべてのグラフの情報には、以下の注意する場合を除き、選択した日付範囲とフィルターが反映されます。 一部のセクションでは、特定のソリューションにフィルターを適用するなど、追加のフィルターを適用することもできます。

## <a name="referrals-summary"></a>紹介の概要

このカードには、リードのパフォーマンスの概要が表示されます。

グラフには、ページ アクセスの総数、アクションの呼び出しボタン (パートナーに問い合わせ) のクリック数、顧客が行動の呼び出しをクリックした後に生成されたリードの数、選択した期間に獲得したリードの合計が表示されます。

変化率メトリック (赤または緑で表示、矢印インジケーター付き) は、**選択した日付範囲の最後の 1 か月間** と **その範囲の最初の 1 か月間** の差を示します。 たとえば、現在の日付が 6 月 15 日で、過去 3 か月間のデータを表示する **[3M]** フィルターを選択したとします。 この場合、これらのメトリックでは、5 月 (選択した期間の最後の 1 か月間) と 3 月 (選択した期間の最初の 1 か月間) の差が示されます。選択した日付範囲は過去 **[3M]** であり、5 月のデータと 3 月のデータの比較になります。

:::image type="content" source="images/referrals/leadsanalyticssummary.png" alt-text="リード分析の概要カードを示す画像。":::

## <a name="conversion-funnel"></a>コンバージョン ファネル

このセクションには、ライフ サイクルを通じた案件の状態の変化が表示されます。 このセクションのメイン ピボットに基づいて、取引量ベースと取引額 (USD) ベースの両方でライフサイクル全体を表示できます。 最初のセクションには、取引の種類のラベルが付き、ボリュームまたは値を種類別に視覚的に示します。 また、過去のリードというセクションがあります。これは、それらを受け入れる、または下げ、またはレポートに対して選択された期間に獲得/失われたとしてマークするアクションを実行した取引を示すために使用されます。 フィルターを適用して、ライフ サイクルのさまざまな段階を通じた案件の進行状況を表示できます。

:::image type="content" source="images/referrals/leadsanalyticsfunnel.png" alt-text="紹介のコンバージョン ファネルを示す図。":::

## <a name="leads-by-geography"></a>地域別のリード

このセクションには、案件の国/地域と、それぞれの国/地域の詳細が表示されます。 案件の詳細を国ごとに示すテーブル ビューと、すべての国のマップ ビューが表示されます。 テーブル内の特定の国をクリックするかマップ ビュー上をクリックすると、特定の国にズームできます。

:::image type="content" source="images/referrals/leadsanalyticsgeodistribution.png" alt-text="紹介の地理的分布を示す図。":::

## <a name="leads-by-program"></a>プログラム別のリード

このテーブルとグラフの組み合わせにより、最も多くの紹介と最高の取引価値を推進している紹介プログラムのリードを確認できます。
このテーブルには、合計潜在顧客が含まれています。また、SLA (24 時間) で受け入れられた潜在顧客は、パーセンテージと絶対期間の両方で、期限切れになった潜在顧客と、"受注" になった潜在顧客と、"USD currency" で受注した潜在顧客の合計値が返されます。 また、選択したプログラムに基づいて、テーブルの右側に潜在顧客の傾向グラフも表示されます。このグラフには、取引数の合計と、取引金額が米国ドルで受注した金額が表示されます。 既定では、すべてのソリューションが選択されています。

:::image type="content" source="images/referrals/leadsanalyticsreferralsprogram.png" alt-text="修飾された参照プログラムを示す画像は、パフォーマンスを向上させるものです。":::

## <a name="declined--lost-reasons"></a>辞退の理由と失注の理由

このセクションでは、潜在顧客が会社によって **拒否** または **紛失** とマークされた理由を分析する方法について説明します。 これらの表記のオプションは、販売元が、拒否または損失として潜在顧客をクローズするときに選択したのと同じ理由です。

:::image type="content" source="images/referrals/leadsanalyticsreasons.png" alt-text="潜在顧客が紛失したときにパートナーによって選択された理由を示す画像。":::

## <a name="comparison-charts"></a>比較グラフ

比較セクションでは、ボリューム上の複数のディメンションに基づいて潜在顧客に関連するデータを比較し、USD pivot で潜在顧客の受注額を比較することができます。
次の 3 つのディメンションから選択して、データを比較することができます。

- 修飾の種類
- 市場
- 紹介プログラム

[修飾の種類] が選択されている場合は、marketplace の潜在顧客と見込みのある潜在顧客に関して、の紹介のパフォーマンスを比較するオプションがあります。 市場と紹介プログラムの両方について、最大3つの異なるオプションを選択してパフォーマンスを比較できます。 1 番目のグラフ (縦棒グラフ) には、メイン ピボット (受注量または受注額) に基づいて月ごとのトレンドでデータが表示されます。 縦棒グラフの右側には円グラフも表示されます。このグラフには、同じデータに対する分布がパーセント表示されます。

:::image type="content" source="images/referrals/leadsanalyticscompare.png" alt-text="比較セクションを示す図。":::

## <a name="no-data"></a>データなし

次に示すように、潜在顧客分析にアクセスするときに、次のような空のグラフを取得する理由は複数あります。

- このアカウントにはデータが存在しない。 このレポートが事前設定されるように案件を試しに作成した。
- ネットワーク接続に何らかの問題がある。 インターネット接続を確認してから、もう一度お試しください。
- 適用したフィルターに一致するレコードが存在しない。 フィルターをリセットしてみてください。
- 潜在顧客の状態が変化してから、分析レポートで同じものが更新されるまでの間には遅延があります。 24 時間後にレポートをご確認ください。

:::image type="content" source="images/referrals/nodata.png" alt-text="潜在顧客のデータの視覚化が表示されていないイメージ。":::
