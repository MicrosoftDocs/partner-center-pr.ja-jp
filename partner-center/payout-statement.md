# <a name="payout-statements"></a>支払い明細

**適切なロール**: Account admin |全体管理者

**支払い明細書** には、商用マーケットプレースを通じて販売されたプランからの支払いの概要が示されています。 収益のトランザクション履歴が表示され、次の支払いが推定され、支払いの傾向が表示されます。 トランザクション履歴と支払い明細書をダウンロードすることもできます。 この記事では、支払い明細書にアクセスする方法について説明します。また、パートナーセンターでは、さまざまな支払いページとダウンロードにアクセスできます。

>[!NOTE]
>MPN Id と関連付けられているプログラムのデータのみが表示されます。 追加のデータを表示する場合は、アカウント管理者に連絡してアクセス許可を確認してください。

### <a name="transaction-history-summary"></a>トランザクション履歴の概要

このビューには、製品の販売実績の日付、状態、および推定月を含む、前払いの詳細が表示されます。

:::image type="content" source="images/payouts/transaction-history.png" alt-text="トランザクションの履歴。":::

- [**獲得日**] –購入日。
- **収益の** 種類 – 収益の種類 (販売、支払い、コーオペなど)。
- **合計金額** – 純収益額。 コマーシャル マーケットプレースでは、これは、標準のマーケットプレース料金を差し引いた後を意味します。
- **状態** – 次の 3 つのオプションがあります。
    - **今後 –** 収益は保留中の冷却期間中です。
    - **処理 –** 収益は次の支払いのために準備されます。
    - **送信** 済み – 収益が支払われた。
- **推定支払月** – 収益の支払いが予想される月。 詳細については、 [次のセクション](#estimated-payment-month) を参照してください。

トランザクションが支払い資格を満たすと、収益トランザクションが表示されます。 収益が不足している理由や予期しない収益がある理由については、「コマーシャル マーケットプレースの支払いに関する一般的な [質問」を参照してください](payout-faq.md#why-are-my-earnings-missing)。

#### <a name="estimated-payment-month"></a>推定支払月

[トランザクション履歴] ページに、次の数か月間の推定支払額を示すテーブルが表示されます。 この情報は、[トランザクション履歴] および [概要] レポートのエクスポートで表示およびダウンロードできます。 この情報により、調整と支払い予測が簡単になります。

推定支払月は、プログラム構成ルールとタイムラインに基づいて計算され、次または今後の支払いサイクルで処理されます。

支払い月の見積もりは、現在、生協を除くすべての収益の種類で使用できます。これは[適用外] **として表示されます**。 2020 年 7 月 1 日より前の収益の場合、[推定支払い月] は [利用できません] **と表示されます**。

次の表は、推定支払月の例を示しています。

| 月 | Amount |
| ------ | :-----------: |
|  Sep-2020 |  $7,273.99   |
|  Oct-2020 | $8,692.30  |
|  2020 年 11 月 | $107.89  |

推定金額は、さまざまな理由で実際の金額とは異なる場合があります。

- 収益の修正: 収益が再計算された場合、実際の金額は異なります
- 調整: 実際の金額は、発生または送信された調整によって異なります。
- ルールの変更: ルールの変更は、実際に支払われた金額の再計算を反映している可能性があります
- 支払い: 支払いの失敗が発生した場合、実際の金額が異なる可能性があります

支払いは、プログラムのしきい値と支払い適格性ルールが満たされている場合にのみ、予定月にリリースされます。 これらのルールには、以下の一覧が含まれますが、これらに限定されません。

- 税プロファイルが最新である必要があります
- 収益は、プログラム ガイドで定義されている最小収益しきい値を満たすか、超える必要があります。
- 保留時の支払い: プロファイルの割り当てページで [支払いを保留する] オプションを選択した場合。
- 支払い方法を使用できない: 支払いまたは税金プロファイルが完了していない。

## <a name="payments"></a>支払い

[ **支払い]** ページには、Microsoft で獲得した金額の詳細が表示されます。 また、支払いを受け取る時間と金額も表示されます。

>[!Note]
> 支払いの対象となるには、収益が $50 の[支払いしきい値](payment-thresholds-methods-timeframes.md)に達している必要があります。 詳細については、「契約」[をMicrosoft Publisherしてください](/legal/marketplace/msft-publisher-agreement)。

:::image type="content" source="images/payouts/payments-overview.png" alt-text="支払いの概要画面。":::

- **今年の支払** い合計 – すべてのプログラムについて、今年の合計支払額 (米国ドル)。
- **次の支払い** 見積もり – 1 回の次回の支払い (近日他の支払いがある場合でも)、米国ドルで行います。
- **最終支払** い – 最新の支払いの金額 (米国ドル)、プログラム名、プログラム。
- **ソース別の** 支払い – 過去 12 か月間のプログラムごとの支払額 (米国ドル単位)。

- **有料** – すべての支払いが正常に送信されました。 ドロップダウン メニューで年を選択して、その年にリリースされた支払いをフィルター処理します。
- **保留中** – 今後の支払い。
- **サービス料金税 (PDF フォーム)** – サービス料金税の対象となる支払いで利用できます。 サービス料金の税は、 **その他の税金** で示されています。
- **View** –支払いに含まれる収益の一覧を使用して、トランザクション履歴にリダイレクトします。

売上が不足している、または予期しない利益が生じる理由を理解するには、「 [商用 marketplace 支払いに関する一般的な質問](payout-faq.md#why-are-my-earnings-missing)」を参照してください。

| 税金請求書の審査中 | 税金請求書を審査中です。 税金請求書が承認されると、支払いが行われます。 | いいえ |

## <a name="export-data"></a>データのエクスポート

[ **データのエクスポート** ] ページは、それ自体では更新されません。 最新のデータを表示するために、ページを手動で更新しなければならない場合があります。 3つのタブから選択して、 **トランザクション履歴**、 **支払い**、 **トランザクションの概要**、または過去の **ステートメント** のいずれかをエクスポートします。

フィルターを使用すると、 **データが使用できない** というエラーが発生する可能性があります。 これは、既定の期間を3か月で選択したままにしてから、その期間外の前払いの支払い ID を選択した場合に発生する可能性があります。 この問題が発生した場合は、期間を拡大して、もう一度やり直してください。

### <a name="historical-statements"></a>履歴明細書

また、[ **データのエクスポート** ] 概要では、履歴ステートメントにアクセスすることもできます。

> [!NOTE]
> 履歴ステートメントはスナップショットであり、更新されません。 サポートに問 [い合わせ](https://partner.microsoft.com/support/v2/?stage=1) 、必要に応じて最新のデータを要求してください。

:::image type="content" source="images/payouts/pc-export-statements.png" alt-text="履歴ステートメントをエクスポートします。":::

- 2019 年 7 月 1 日より前のトランザクション履歴は個別に処理され、後の履歴レポートとは異なるフィールドが使用されます。
- レガシ トランザクション履歴には"予約済み" という列があります。これは、最新の履歴の "収益" 列に対応します。ただし、状態が "Payment Sent" と等しいすべての収益は除外されます。
- 3M、6M、または 12M などのフィルターは、履歴明細書セクションには適用されません。
