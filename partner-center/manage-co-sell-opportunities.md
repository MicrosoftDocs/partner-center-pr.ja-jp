# <a name="manage-co-sell-opportunities-in-partner-center"></a>パートナーセンターで共同販売の機会を管理する


**適切なロール**: MPN partner admin |参照管理者

共同販売の機会を活用しましょう。  共同販売の機会は、顧客のニーズを満たす製品を販売するための、複数の当事者のコラボレーションです。 この記事では、パートナーセンターで共同販売の機能を使用して、新しい顧客を効果的に共同販売および取得する方法について説明します。

## <a name="types-of-co-sell-opportunities"></a>共同販売の機会の種類

**共同販売の機会** -1 人以上のマイクロソフトの営業担当者が、お客様の問題を解決するために積極的に関与しています。 営業案件は、Microsoft の営業担当者がパートナーに共同販売活動に参加するように招待している場合や、取引を終了するために Microsoft sales からの支援を必要とするパートナーとの間で発生する可能性があります。

パートナー **ツーパートナー (P2P)** –1つのパートナーが Microsoft 共同販売エコシステムの他のパートナーを招待して共同作業を行うことができます。 パートナーは、必要に応じて Microsoft の営業担当者を招待して、商談を終了することができます。 パートナーと Microsoft はどちらも、取引を作成した後でも、完了前に招待できます。 (完了には、"勝ち"、"lost"、"拒否"、および "期限切れ" が含まれます)。

**プライベート取引** -パートナーがパートナーセンターで作成された取引に対して個別に作業することを決定した場合。 パートナーは、Microsoft の営業チームに可視化したり、取引を作成した後にアクティブなヘルプを求める Microsoft 販売者を招待したりすることができます。 この案件の詳細は、タイムパートナーが明示的に [はい] を選択するまで、Microsoft のセールスチームと共有されません。 または、Microsoft のセールスチームにお問い合わせください。質問のオプションの1つを選択してください。「Microsoft から必要なヘルプの種類を特定する」を参照してください。

**パートナー主導の** 取引 - これらは、パートナーが個別に取引に取り組む決定を行う取引ですが、Microsoft の営業チーム メンバーに取引の詳細を表示しても問題ありません。 パートナーは、取引の作成後に Microsoft 販売者を招待できます。 これらの取引は、インセンティブ対象ソリューションが取引の一部である場合に、取引登録の対象となります。 インセンティブ対象のソリューション条件が満たされている場合、取引登録に Microsoft 販売者を招待する必要はありません。 取引パートナー主導の取引を行う場合は、「Microsoft の販売者にこの取引を表示しますか?」という質問に対して「はい」と答えます。

**ソリューション評価 (SA)** – これらの種類の取引は、Microsoft テクノロジを使用または使用する計画を行っているお客様に対してテクノロジ評価を実行するために、ソリューション評価ビジネス チームによって審査された一連のパートナーだけが作成できます。

> [!NOTE]
> プライベート取引は、真にコラボレーションされていない場合でも、共同販売の機会タブに含まれています。これは、任意のパイプライン取引を、任意の非ターミナル ステージ **(獲得**、失い、拒否、期限切れ) の共同販売機会に簡単にアップグレードするパスを提供します。

## <a name="navigating-the-co-sell-section"></a>[共同販売] セクションの移動

共同販売セクションにアクセスするには、次の手順に従います。

1. パートナー センター [ダッシュボード](https://partner.microsoft.com/dashboard)にサインインします。

2. 左側のナビゲーション メニューから、[紹介] を **選択し**、続いて [共同販売の機会 **] を選択します**。

   このセクションには、次の 3 つのトップ レベルのタブが表示されます。

   - **受信 –** 共同販売のために Microsoft 販売者からパートナーに送信される機会はすべて、このタブで使用できます。

   - **送信 –** パートナーによって作成された機会はすべて、このタブで使用できます。パートナーは、共同販売、プライベート、パートナー間の 3 つの異なるカテゴリの機会を作成できます。

   - **[お気** に入り] – [受信] または [アウトバウンドの共同販売案件] タブでお気に入りとしてマークされた機会が、このタブに表示されます。リスト ビューで営業案件の中心アイコンをクリックして、任意の営業案件をお気に入りとしてマークします。 同じアイコンをもう一度クリックするだけで、お気に入りから営業案件を削除できます。

## <a name="creating-a-co-sell-opportunity"></a>共同販売の機会を作成する

> [!IMPORTANT]
> Microsoft と共同販売する場合、パートナーは、販売可能なソリューション、連携する販売チーム、および強力な市場投入戦略が必要です。 さらに、特定の追加条件も満たす必要があります。 詳細については、「 [共同販売を準備](https://partner.microsoft.com/reach-customers/selling-with-microsoft#become-ready)する」を参照してください。

3種類の営業案件を作成できます。

### <a name="select-your-customer"></a>顧客を選択する

特定の取引の顧客を選択します。 場所を入力し、名前の入力を開始します。 [候補] ボックスに、検索条件に基づいて推奨事項が表示されます。 多くの企業では同じ場所に複数のビジネスエンティティがあるため、複数の結果が表示される場合があります。 どちらを選択すればよいかわからない場合は、顧客にどのような D-u-n-s 番号が属しているかを確認できます。 作業中の顧客と完全に一致するものを選択してください。

[ **顧客の選択**] をクリックすると、この案件の主要な連絡ポイントであるユーザーの名前、電話番号、および電子メールアドレスを入力するように求められます。 これらの詳細はオプションであり、Microsoft の販売を招待してエンゲージメントを支援する場合にのみ必要です。 **[次へ]** を選択します。

> [!IMPORTANT]
> お客様の連絡先情報をマイクロソフトに提供することについて、お客様の同意を得ていることを確認してください。 Microsoft は、この情報を使用して顧客に直接問い合わせることができます。

### <a name="deal-details"></a>案件の詳細

現在の取引の詳細を入力します。 以下で定義されているフィールドは、顧客と積極的に連携して取引を終了するために変更できます。

| **フィールド名** | **必須/省略可能** | **詳細** |
|-------------|--------|-------|
|**取引名** | Mandatory | 後で取引を識別するための表示名。 |
|**場所**| Mandatory | 紹介の MPN 場所スコープ。 この場所のスコープを持つ紹介ユーザーは、チームの一部である場合に紹介を表示できます。 グローバル スコープを持つ紹介管理者と紹介管理者は、場所に関係なく紹介を表示できます。 紹介を作成した後は、場所を編集できません。|
|**推定値** | Mandatory | 取引の作成時に使用可能な情報に基づく取引の値。|
|**推定終了日**| Mandatory| 顧客との取引を終了する日付。 |
|**CRM ID**| 省略可能 | 取引に、追跡目的でそれぞれの CRM の営業案件の ID をタグ付けします。|
|**マーケティング キャンペーン ID**| 省略可能 | 取引の結果として生み出されたマーケティング キャンペーンをキャプチャします。 この登録は、同じ ID でキャンペーンから発生した取引にタグを付ける場合に、特定のキャンペーンの ROI を追跡するのに役立ちます。|
|**メモ**| 省略可能 | すべての最新情報を更新して、同じ取引に取り組む会社の他の従業員を可視化したり、取引の現在の状態を把握したりします。 これは、Microsoft 販売者や他のパートナーと会社との間で話し合う場合の記録上のコミュニケーションとして使用できます。|

### <a name="add-team-members"></a>チーム メンバーを追加する

取引の詳細を追加した後、この特定の取引に取り組む従業員を追加します。 従業員の名前、電話番号、電子メール アドレスを入力する必要があります。 これらの詳細は必須であり、取引を作成するには、少なくとも 1 人の連絡先とすべての詳細を入力する必要があります。 これらの詳細は、取引を作成した後でも変更できます。 前の取引からの最近の連絡先は、すぐに取引に追加するために右側に表示されます。 P2P 取引の場合、チームは、招待状を送信する会社と会社の両方の従業員を持つことができます。

### <a name="add-solutions"></a>ソリューションの追加

このセクションでは、この処理の一部となるソリューションに関連する情報を提供する必要があります。 これは、取引を作成するために少なくとも1つのソリューションを追加する必要がある必須セクションです。 ソリューションの詳細は、取引を作成した後に変更できます。 次に説明するように、複数の種類のソリューションを取引に追加することができます。

- **自分の会社のソリューション:** これらは、会社によって公開されている共同販売可能なソリューションです。
- **Microsoft:** これらのソリューションは、Microsoft によって所有されています。
- **その他のサードパーティ製ソリューション:** これらは、Microsoft 共同販売エコシステムの他のパートナーによって発行された、共同販売の準備ができたソリューションです。
- **ソリューションの評価:** これらは評価の種類であり、資格のあるパートナーは顧客のニーズに基づいて選択できます。

> [!Important]
> ソリューション評価に対して選択できる評価の種類は1つだけです。他のソリューションを追加することはできません。 ソリューション評価を選択すると、パートナーは、評価を作成する場所を選択する必要があります。 これは、適切なインセンティブ支払いを行うために必要です。

ソリューション情報を指定したら、[次へ] を選択して、販売の種類を決定できるセクションに移動します。 ソリューションの評価ではなく、ソリューションの種類の最初の3つのオプションからソリューションを選択した場合は、次の4つのオプションがあります。

**プライベート取引**: この手順で Microsoft に招待してエンゲージメントを作成しない場合は、プライベートパイプラインの種類になります。 Microsoft の販売者は、この問題の詳細を把握することはできません。 プライベート取引を作成するには、**この時点で** **[Microsoft が必要とするヘルプの種類を指定** してください。] 質問については、 **「Microsoft の販売者にこの商談を見** たいと思いますか?」という質問が **あり** ます。 取引が終了状態 (獲得、失われた) に達する前に、いつでもプライベート取引をパートナー主導またはアクティブな共同販売取引にアップグレードできます。

> [!Important]
> 取引登録は、プライベート取引には適用できません。 インセンティブ対象ソリューションを使用してプライベート取引を作成する際は注意が必要です。これらのソリューションは、パートナー センター で取引登録の対象となパートナー センター。

**パートナー主導の** 取引: Microsoft 販売チームから積極的な支援を求めず、パイプラインを可視化したい場合は、パートナー主導の取引を作成できます。 パートナー主導の取引を作成するには、"Microsoftから必要なヘルプの種類を特定する" という質問に対して[この時点でヘルプは必要ありません]**を選択します。** 「Microsoft の販売者にこの取引を表示しますか?」という質問に対して[ **は** い] **と [はい] を選択します**。 これらの取引は、Microsoft 販売者がパートナーの支援に積極的に関与していない場合でも、取引登録の対象となります。 取引登録の要件は、取引のインセンティブ対象ソリューションを持ち、取引の獲得状態です。

**共同販売取引:****"Microsoft** から好きなヘルプの種類を特定する" という質問に対して既定の選択以外のオプションを選択した場合、この取引は共同販売取引に変わるので、Microsoft 販売者が取引を終了するのに役立つ可能性があります。 Microsoft からの支援を要求しても、Microsoft 販売者が取引に参加する保証はありません。 Microsoft の営業担当者は、参加を決定するために 14 日間を持っています。 [メモ] セクションで、必要なヘルプの種類を必ず特定してください。

**パートナー対パートナー (P2P)** の取引: [パートナーの招待] リンクをクリックして、他のパートナーを取引に招待できます。 P2P 取引を作成するプロセスを次に示します。

- **パートナーを選択します。** [パートナーの招待] をクリックすると、パートナー名の入力を開始して、入力する名前に一致するパートナーの候補一覧を取得できます。 関心のあるパートナーを選択して、そのパートナーの詳細を入力します。 Microsoft 共同販売エコシステムに属し、パートナーセンターで拘束力しているパートナーだけを検索できます。

- **推定終了日:** これは、招待されたパートナーが取引の一部を完了する予定の日付です。 日付は事前に入力されているので、必要な場合にのみ日付を変更できます。 これは必須フィールドであり、取引を作成した後に招待するパートナーによって編集できます。 取引を作成した後に、このフィールドを変更することはできません。

- **推定値と通貨:** これは、招待されたパートナーが全体的な取引に参加する処理の価値です。 招待されたパートナーが取引の一部であるかどうかを判断できるように、正しい値をここに入力してください。 招待されたパートナーは、取引を作成した後でこの値を変更できます。 取引を作成した後に、このフィールドを変更することはできません。

- **メモ:** この取引の一部としてパートナーを招待する理由の詳細を追加します。 詳細情報は、招待されたパートナーが参加を希望するかどうかを判断するのに役立ちます。

- **チームを追加します。** 招待されたパートナーと協力する従業員を会社から追加します。 招待されたパートナーが商談を受け入れた場合は、両方の企業がチーム全体のビューを使用できるように、自分の従業員を追加することができます。 これらの詳細は、取引を作成する前にのみ変更できます。 取引データに入力された従業員の詳細は、この特定のパートナーと協力している従業員を簡単に選択できるように、事前に入力されています。

- **ソリューションの追加:**  招待されたパートナーがテーブルに取り込むソリューションを追加します。 少なくとも1つのソリューションが必須です。 招待されたパートナーは、招待を受け入れるとソリューションを変更できます。

- **ヘルプの種類を特定します。** ヘルプの種類を特定します。最後に、招待されたパートナーから必要な特定のヘルプを特定します。

この処理に参加するすべてのパートナーに対して、この手順を繰り返します。 パートナー間の取引では、Microsoft とパートナーの両方を取引に招待する Microsoft 販売者も関与できます。 また、取引を作成した後で、Microsoft とパートナーの両方を招待することもできます。

## <a name="responding-to-a-co-sell-opportunity"></a>共同販売の機会への対応

各機会は、独自のライフ サイクルを通じて移動します。

### <a name="received-stage"></a>受信したステージ

このステージでは、Microsoft 販売者または Microsoft 共同販売エコシステムの他のパートナーから新しい共同販売の機会を受け取った場合は、詳細を確認し、ビジネス ニーズについて詳しく知りたい場合は、お客様にお問い合わせください。 このステージでは、2 つのアクションを実行できます。 紹介を受け入れるか拒否します。

- **受け入れ:** 取引の名前を入力し、レビューに基づいて推定取引額と推定購入期間を編集します。 顧客との連絡先を確立したら、[メモ] フィールドに情報を入力して、顧客が探している内容の詳細を説明する必要があります。 必要に応じて、ここに CRM ID (参照用のみ) を入力し、それぞれの営業案件を生み出したマーケティング キャンペーン ID を入力し、この取引に取り組む会社の連絡先を追加することができます。

- 完了したら、**[次へ]** をクリックします。 紹介を次のステージに移します。つまり、顧客のニーズに対応するために積極的に関与する予定です。 また、この情報は、今後同様の取引を見つけるのに役立ちます。

- **[** 拒否] : 取引を拒否する理由を選択し、含めるメモを追加してから、[取引を終了] **を選択します**。 Microsoft は拒否済みとしてアーカイブ **し** 、この機会を送信した Microsoft またはパートナーに通知します。

- 割り当てられた時間 (現在は 14 日) 内に応答しない場合は、期限切れとしてアーカイブし、この機会を送信した Microsoft またはパートナーに通知します。

### <a name="accepted-stage"></a>受け入れ可能なステージ

お客様との取引の成約に向けて作業します。 受け入れられる紹介に対して指定した情報を変更する場合は、[編集] を選択 **します**。 その後、取引名、推定購入日、推定値、メモ、CRM ID、マーケティングキャンペーン ID を更新できます。  また、[ **チームの追加** ] を選択して、取引に従事している他のユーザーの名前、電話番号、電子メールアドレスを指定することもできます。 ソリューションは、お客様のニーズに基づいて編集することもできます。

既定では、作成したすべての取引が "承認済み" 段階にあります。

処理を開始した後は、取引のライフサイクルの販売ステージをマークすることによって、進行状況の詳細を指定できます。 次に示すように、最初の受け入れまたは作成と、最終的に発生したまたは失われたステージとは別に、取引のライフサイクルには4つのステージがあります。 これらの詳細情報は省略可能ですが、共同販売の担当者からの適切な支援を得るために、これらの情報を共有することを強くお勧めします。

> [!Note]
> 営業段階は、ソリューションの評価に対処する場合に異なります。 ソリューション評価では、販売ステージのマーキングも **必須** です。 [**勝ち**] ボタンは、すべての販売ステージがパートナーによって完了とマークされた後にのみ有効になります。

次に示すのは、Microsoft パートナーセンターの紹介システムによって決定されたソリューション評価以外の取引の営業段階と対応するパーセンテージを示しています。

|**販売ステージ名**|**営業段階の割合**|**営業段階の定義**|
|:----|:-----|:-----|
|作成済み|10%|送信取引を作成する。|
|同意|10%|受信処理の受け入れ。|
|資格のある|20%|さらに進める前に、取引と顧客の要件の価値を見極めます。|
|確立|40%|詳細な要件を理解し、POC または正式な提案に必要なその他のアーティファクトを準備する方法について詳しく説明します。|
|提案|60%|要件に基づいて顧客に正式な提案を行う。|
|ネゴシエート|80%|最終的な状態を得る提案に基づいて最終的な条件をネゴシエートする 。|
|Won|100%|取引を獲得としてマークする。|

完了したら、2 つのアクションのいずれかを実行できます。このアクションは、取引を Won または **Lost** としてマークして結果を報告します。

会社が同じ販売ステージに従う必要はありません。 API を使用してこれらの値を渡す場合、パートナー センターは取引の販売ステージを認識し、会社のステージをこれらの標準ステージに自動的にマップする方法です。 UX の一部パートナー センター使用している場合は、表に示されている割合を使用して販売ステージをマークします。

> [!Note]
> 特定の適格なソリューションの場合、Won を選択した後、取引を登録するために追加情報を提供する必要があります。 ここで入力した情報は Microsoft によってレビューされ、レビュー プロセス中に追加情報を求められる場合があります。 詳細については、「[取引の登録](register-deals.md)」を参照してください。

> [!Important]
> 取引が取引登録の対象となる場合は、取引の作成と取引の間に 72 時間のギャップが生じ、獲得済みとしてマークされます。 それより前に獲得した取引を終了すると、取引登録が拒否される可能性があります。

取引は、以下のすべての条件を満たしている場合にのみ、取引登録の対象となります。

- 取引の種類は共同販売またはパートナー主導です。
- この取引にはインセンティブ対象のソリューションがあります。
- 取引の状態が獲得します。
- 取引の種類が共同販売の場合、Microsoft は招待を受け入れるか、取引を獲得済みとしてマークしました。 Microsoft の状態を理解するには、取引の詳細の下にある Microsoft カードを参照してください。

> [!Important]
> 取引の会社名とインセンティブ対象ソリューションが顧客との契約で明確に記載されている場合にのみ、取引を登録します。

取引が取引登録の対象となる場合は、次に示すように、取引のライフサイクルに "取引登録" と呼ばれる別のマイルストーンが追加されます。

:::image type="content" source="images/pscmigration/dealregstages.png" alt-text="取引のライフサイクル、取引登録を開始できる場所を示す画像。":::

商談を受注済みとしてマークした後、または後で [商談ライフサイクル **登録** ] ボタンを使用して、すぐに取引を登録することを選択できます。
取引が登録されると、同じライフサイクルから取引検証の進行状況を確認できます。 会社から何らかのアクションが必要な場合は、該当するエラーが [取引のライフサイクル] ビューに表示されます。 取引の検証が完了すると、取引は closed 状態になります。

> [!Important]
> 取引レビューと最終的な検証の状態は両方とも、IP 共同販売取引にのみ適用されます。

### <a name="combinations"></a>組み合わせ

次の表は、この段階で招待できるユーザーの組み合わせを示しています。

|**元の取引の種類**|**招待できるユーザー**|**メモ**|
|-----|:-----|:-----|

|プライベート|取引を表示する Microsoft 営業チーム|Microsoft の営業チームが取引を可視化できるようになると、取引はパートナー主導にアップグレードされます。|
|パートナー主導|Microsoft およびその他のパートナー|Microsoft が招待されている場合、取引は共同販売にアップグレードされます。|

|Microsoft を使用せずにパートナーからパートナーへ|その他のパートナー||

### <a name="closed-stage"></a>終了ステージ

これは、すべての機会の最終段階です。 クローズ ステージで、獲得、失われた、拒否された、期限切れのすべての **取引** を表示できます。 このステージでは実行できるアクションはありません。

## <a name="frequently-asked-questions"></a>よく寄せられる質問

**Q。取引は、獲得または失われたとマークされた後に編集できますか?**

いいえ。取引は、一度ターミナル状態に移行した後は変更できません。 期限切れ、拒否、獲得、および失われたのは、取引に対してそれ以上の更新が可能ではないターミナル状態です。 これらのターミナル状態に取引を移動する場合は注意が必要です。

**Q。電子メール通知を受け取るユーザーはパートナー センター。**

以下のワークフローでは、新しいパートナーの受信紹介について、パートナー センターの紹介システムからパートナーに電子メールを送信する方法について説明します。

:::image type="content" source="images/pscmigration/emaillogic.png" alt-text="新しい受信紹介のためにパートナーに電子メールを送信する方法のロジックを示す画像。":::

**Q。取引登録の対象となるインセンティブの種類は何ですか?**

取引登録の対象となるのは、IP インセンティブのみです。 次のような方法があります。

- Azure IP 共同販売
- Biz Apps Premium
- Biz アプリの標準

## <a name="get-more-co-sell-opportunities"></a>共同販売の機会を増やす

ここでは、お客様のビジネスに適した共同販売の機会を得るために役立つヒントを紹介します。

- **商談に迅速に対応** します。 受信要求に対してタイムリーに応答すると、今後のパートナー検索結果の表示が徐々に向上します。 チームが意向を迅速に応答できるように体制を整えてください。
- **承諾する取引を慎重に選び** ます。 同意して拒否した取引の種類を監視し、この情報を使用して類似の取引を見つけることができます。 適切に適合していない取引を受け入れると、検索結果が改善されず、受信した営業案件の品質に影響を与える可能性があります。
- **取引規模の見積もり、予定終了日、最終的な契約の成否 (受注または失注) をマイクロソフトに報告してください**。 この情報を使用して、引き続き品質の紹介を提供します。
