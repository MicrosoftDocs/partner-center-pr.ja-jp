# <a name="explore-the-azure-and-cloud-products-benefits-area-in-partner-center"></a>パートナーセンターで Azure とクラウド製品の特典領域を調べる

**適切なロール**

- グローバル管理者
- ユーザー管理者
- 管理エージェント
- MPN パートナー管理者

この記事では、パートナーセンター内からパートナーが利用できる特定の Azure およびクラウドの特典について説明します。 また、特定の種類の Azure サブスクリプションとクラウドサブスクリプションをアクティブ化する方法についても説明します。
 
## <a name="overview"></a>概要

Microsoft Partner Network のパートナーとして、さまざまな分野の多くの技術および市場投入の特典にアクセスできる場合があります。 パートナーセンターで上記のいずれかのロールが割り当てられている場合は、パートナーセンターのダッシュボードから直接、これらの利点の多くを調べることができます。

> [!NOTE]
> パートナーセンターの特典領域にアクセスするには、パートナーセンターの [ダッシュボード](https://partner.microsoft.com/dashboard) にサインインし、左側のナビゲーションメニューから [ **MPN** ] を選択する必要があります。 次に、メニューから [ **特典** ] を選択し、[特定の特典] オプションまたは [領域] を調査します。

利用できる利点の1つとして、 **Azure & cloud 製品** があります。 パートナーセンターの **特典** メニューから [ **azure & cloud** ] を選択すると、[ **azure & クラウド製品**] ページが表示されます。

このページには、次の詳細が表示されます。

- ライセンスベースのサブスクリプションまたは使用量ベースのサブスクリプションにかかわらず、さまざまな Azure & クラウド製品サブスクリプション特典をご利用いただけます
- 各製品に割り当てられたライセンス数
- 特定の特典製品をプロビジョニングしたオファーの種類
- 特典の種類 (コアかインクリメンタルかにかかわらず)
- 特定の特典が有効期限切れになった日付

:::image type="content" source="images/benefits/mpn-benefits-azure-cloud-1.png" alt-text="Azure とクラウド製品のページには、ご利用いただけるサブスクリプション特典が掲載されています。" border="false":::

## <a name="usage-based-vs-license-based-subscriptions"></a>使用量ベースおよびライセンスベースのサブスクリプション

**Azure & クラウド製品** のページには、次の2種類のサブスクリプション特典が用意されています。
- 使用量ベースのサブスクリプション (Microsoft Azure クレジット) 
- ライセンスベースのサブスクリプション (Office 365 など)

### <a name="usage-based-subscriptions-with-two-types-of-microsoft-azure-credit"></a>2種類の Microsoft Azure クレジットを持つ使用量ベースのサブスクリプション

使用量ベースのサブスクリプションには、Microsoft Partner Network メンバーシップで利用できるアクティブプランに応じて、2種類があります。 このようなアクティブプランには、Microsoft Action Pack、シルバーコンピテンシー、gold コンピテンシーなどがあります。 

これらのプランに対してプロビジョニングできる使用量ベースのサブスクリプションには、 *azure の月単位のクレジット* や *azure 一括 (毎年) クレジット* の2種類があります。

- **Azure の月額クレジットのプロビジョニング:** これは、$ 100/month (または現地通貨で同等) のように表示される場合があります。<br/>
*and/or*
- **Azure 一括 (毎年) クレジットのプロビジョニング:** これは、$ 6000/年、$ 12000/年、または $ 13000/year (現地通貨で同等のもの) のように表示される場合があります。

パートナーセンターで Microsoft Azure サブスクリプションをアクティブ化するには、社内 (またはゲストユーザー) 内のユーザーにアクセス権を割り当てる必要があります。 ユーザーは、Azure Active Directory テナントにユーザーエントリを既に作成しておく必要があります。 その後、次の操作を行います。

1. ユーザーが Azure AD テナントにユーザーエントリを持っていることを確認したら、パートナーセンターの [ダッシュボード](https://partner.microsoft.com/dashboard)にサインインします。

1. 左側のナビゲーションメニューで [ **MPN** ] を選択し、[ **特典**]、[ **Azure と cloud**] の順に選択します。

1. アクティブ化する使用状況に基づくサブスクリプションの [ **詳細** ] 列の下矢印を選択します。 

1. パートナーセンターの **Azure とクラウド製品** のページ (下記) で、[**ユーザーの割り当て**] 領域を探します。 次に、[ **ユーザーの割り当て** ] オプションを選択して、目的の使用状況に基づいたサブスクリプション特典をそのユーザーに割り当てます。 

   :::image type="content" source="images/benefits/mpn-benefits-azure-cloud-2.png" alt-text="[Azure とクラウド製品] ページ。ユーザーの割り当て領域が表示されます。" border="false":::

- 1か月あたり $100 のような Azure の月単位のクレジット (または現地通貨で同等のもの) を割り当てる場合、このサブスクリプションのユーザーをいつでも追加または削除することができます。

- Azure 一括 (毎年) クレジットを割り当てるときは、パートナーセンターで [ **削除** ] オプションを使用できないことに注意してください。 代わりに、登録年中に他のユーザーに一括クレジットを転送する必要があります。 Azure bulk (年次) クレジットサブスクリプションは、他のライセンス、前払い、またはクレジットプランと組み合わせることもできません (たとえば、Microsoft Enterprise Agreement、Cloud Solution Provider、Azure インオープンプラン、Azure Pass、Microsoft Action Pack など)。

### <a name="license-based-subscriptions"></a>ライセンスベースのサブスクリプション

[Azure とクラウド製品] ページの [ライセンスベースのサブスクリプション] セクションでは、ライセンス認証のためにプロダクトキーを必要とするすべてのクラウド製品 (Microsoft O365、Microsoft Dynamics、Microsoft Intune、Azure AD basic、Power BI など) を確認できます。これらの製品は、Microsoft Partner Network でパートナーとして使用できるアクティブなプランの種類に基づいて提供されます。 このようなプランには、Microsoft Action Pack、シルバーコンピテンシーメンバーシップ、gold コンピテンシーメンバーシップなどがあります。

ライセンスベースのサブスクリプションをアクティブ化するには: 

1. [Azure とクラウド製品] ページで特定のサブスクリプションを見つけます。 

1. 右側にあるドロップダウン矢印を選択し、そのサブスクリプションの [詳細] 列を **表示** します。 

1. 以降のページの指示を参照してください。 これにより、キーを使用する方法の詳細がわかります。 

   :::image type="content" source="images/benefits/mpn-benefits-azure-cloud-3.png" alt-text="詳細を表示した Azure とクラウド製品のページ。" border="false":::


1. キーを利用する準備ができたら、このページの下部にある **アクティブ化チュートリアル** オプションを選択します。
   - 新しいサブスクリプションをアクティブ化する場合、または既存のサブスクリプションを無効または有効期限切れにする場合は、 **プライベートまたは incognito ブラウザー** を開き、次のリンクをコピーして貼り付け、指示に従います。

      [https://signup.microsoft.com/productkeystart](https://signup.microsoft.com/productkeystart)
      
   - 終了日を延長するか、または既存のサブスクリプションに追加のシート/ライセンスを追加する場合は、以下のリンクを **プライベートまたは incognito ブラウザーウィンドウ** に貼り付けて、指示に従ってください。
 
      [https://admin.microsoft.com/AdminPortal/Home#/subscriptions](https://admin.microsoft.com/AdminPortal/Home#/subscriptions)

> [!Note]
> キーが引き換えされると、同じキーをもう一度利用することはできません。 また、次の重要な点にも注意してください。
>
> - ライセンス (シート)、数、終了日を延長するかどうかなどを追加する必要があるかどうかを確認します。 ライセンス (座席) を追加することを選択した場合、終了日を延長し **ない** と、システムによって、アクティブなサブスクリプションが12か月未満であることが示されます。 
>
> - 正しい資格情報でサインインしていること、および正しい Azure Active Directory (AAD) テナントでキーをアクティブ化していることを確認します。 必ず、プライベート/incognito ブラウザーウィンドウでサインインして、正しいテナントでキーをアクティブ化していることを確認してください。
>
> - Microsoft パートナーセンターに表示されている特典の有効期限は、その特典の使用権限を最後に行使した日付を示します。 この日付は、M365 管理センターポータルに表示されているサブスクリプションの終了日とは異なる場合があります。 これは、特典のアクティブ化日、キーに関連付けられているライセンスの数、管理ポータルで追加または拡張されたライセンスなど、いくつかの要因によって発生することがあります。

## <a name="extend-the-end-date-or-add-more-licenses-to-an-existing-subscription"></a>終了日を延長するか、既存のサブスクリプションにライセンスを追加します

次の手順に従って、ライセンスを追加するか、既存のサブスクリプションの終了日を延長します。

1. 次のリンクをコピーして、 **プライベートまたは incognito ブラウザーウィンドウ** に貼り付けます。

   [https://admin.microsoft.com/AdminPortal/Home#/subscriptions](https://admin.microsoft.com/AdminPortal/Home#/subscriptions)

1. **M365 管理センターポータル** にサインインするときは、正しい Azure Active Directory テナントでサインインしていることを確認してください。 次に、一覧から特定のクラウドサブスクリプションを選択し、次のいずれかの操作を行います (サブスクリプションを更新するか、ライセンスを追加します)。

   - 既存のサブスクリプションを更新するには、ライセンス領域の [ **終了日を延長** する] オプションを選択します。 終了日を延長すると、サブスクリプションの期間が長くなり、ライセンス/接続クライアントの数も同じになります。 

   - 既存のサブスクリプションにライセンス (シート) を追加するには、ライセンス領域の [ **ライセンスの追加** ] オプションを選択します。 既存のサブスクリプションを更新する場合は、[ **ライセンスの追加** ] が選択されていないことを確認してください。 [ **ライセンスの追加** ] を選択すると、サブスクリプションの期間が短縮され、サブスクリプションのライセンス数が増加します。

   :::image type="content" source="images/benefits/mpn-benefits-azure-cloud-4.png" alt-text="ライセンスを追加する方法を示す Azure とクラウド製品のページ。" border="false":::

1. サイドパネルで、[ **プロダクトキーの引き換え**] を選択します。 

1. プロダクトキーを入力し、[ **検証** ] を選択して、キーの詳細を確認します。 

1. [ **引き換え** ] を選択してプロセスを完了します。

   > [!NOTE]
   > プロダクトキーを使用すると、プロダクトキーを使用した日から12か月分のサービスを利用できます。 ただし、既存のユーザーが事前更新日を使用している場合は、すべてのユーザーの1つの終了日に基づいてサブスクリプションを管理しやすくするために、2つの日付が平均化されます。 計算中に時間が追加または削除されることはありません。 2つの日付を平均するために使用される計算は次のとおりです。
   >
   > **計算:** \[A (初期ユーザー) x B (サブスクリプションの残り時間) \+ C (新しいユーザー) x (12 か月) \] ÷合計ユーザー数 = 新しい残り時間。

## <a name="activate-a-subscription-for-the-first-time"></a>サブスクリプションを初めてアクティブ化する

サブスクリプションを初めてアクティブ化する場合は、次の手順に従います。 サブスクリプションが無効または期限切れの状態にある場合は、次の手順を使用することもできます。

1. 次のリンクをコピーして、 **プライベートまたは incognito ブラウザーウィンドウ** に貼り付けます。
   
   [https://signup.microsoft.com/productkeystart](https://signup.microsoft.com/productkeystart)
1. 正しい Azure Active Directory テナントアカウントにサインインしていることを確認します。 サインインしたら、テナントの詳細を確認します。

   :::image type="content" source="images/benefits/mpn-benefits-azure-cloud-5.png" alt-text="Azure とクラウド製品のページで、キーを追加する場所が示されています。" border="false":::

1. パートナーセンターから取得した25桁のキーを入力し、[ **次へ**] を選択します。

2. 表示された製品の詳細、ライセンスの数、サブスクリプションの期間を確認します。 準備ができたら、[ **引き換え** ] を選択してサブスクリプションをアクティブ化します。

   新しくアクティブ化されたサブスクリプションは、M365 管理センターの [ **製品** ] タブで使用できるようになります。

## <a name="licensing-scenarios"></a>ライセンスのシナリオ

このセクションでは、Azure とクラウド製品に関連付けられているサブスクリプションのいくつかのサンプルライセンスシナリオについて説明します。 これらのシナリオでは、サブスクリプションの終了日を計算する方法について説明します。

### <a name="scenario-1"></a>シナリオ 1

サブスクリプションで100ライセンスをアクティブにしていて、25ライセンスのキーを使用して終了日を延長しようとしている場合、100ライセンスを持つサブスクリプションでは、終了日が3か月で延長されます。

### <a name="scenario-2"></a>シナリオ 2

サブスクリプションで100ライセンスをアクティブにしていて、100ライセンスを持つキーを使用して終了日を延長しようとしている場合、終了日は12か月で延長されます。

### <a name="scenario-3"></a>シナリオ 3

サブスクリプションで5つのライセンスがアクティブになっていて、10個のライセンスを持つキーを使用して終了日を延長しようとしている場合、終了日は5つのライセンス (またはシート) を持つサブスクリプションに対して24か月延長されます。

### <a name="scenario-4"></a>シナリオ 4

3倍以上のアクティブなライセンス/シートを含むキーを使用して終了日を延長しようとすると、次のエラーメッセージが表示されます。 このメッセージは、サブスクリプションの終了日を最大3年まで延長できるため、仕様によるものです。
