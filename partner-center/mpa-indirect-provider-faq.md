# <a name="how-indirect-providers-can-help-their-indirect-resellers-adopt-the-microsoft-partner-agreement-mpa"></a>間接リセラーが Microsoft パートナーアグリーメント (MPA) を採用するために間接プロバイダーがどのように役立つか

**適切なロール**

- グローバル管理者
- 管理エージェント
- 販売代理店

次の質問と回答では、間接リセラーが MPA に署名する方法と理由を理解するのに役立つ潜在的な問題と解決策について説明します。 

クラウドソリューションプロバイダープログラムの間接プロバイダー、直接請求パートナー、および間接リセラーは、この情報を参考にしてください。

### <a name="what-are-the-legal-implications-if-the-indirect-reseller-does-not-sign-the-microsoft-partner-agreement-mpa"></a>間接リセラーが Microsoft パートナー契約 (MPA) に署名していない場合、どのような法的な影響がありますか? 
 
**回答** (間接リセラーが) MPA に署名しない限り、CSP プログラムで間接プロバイダー (ディストリビューター) を使用することはできません。 MPA は、ビジネスを実行する [CSP リージョン](regional-authorization-overview.md) ごとに署名する必要があります。 間接リセラーが csp リージョン内の任意の間接プロバイダーとの CSP トランザクションの実行に準拠するには、CSP リージョンごとに少なくとも1つの MPA 署名が必要です。 

### <a name="is-mpa-available-in-local-languages"></a>MPA はローカル言語で使用できますか。
 
**回答** 違います。 英語でのみ利用できます。 

### <a name="i-work-with-multiple-indirect-providers-distributors-in-various-countries-do-i-need-to-sign-the-mpa-in-each-csp-country"></a>さまざまな国で複数の間接プロバイダー (ディストリビューター) を使用しています。 各 CSP の国で、MPA に署名する必要がありますか。


**回答** MPN ID に対して、CSP リージョンごとに少なくとも1つの MPA 署名が必要です。 たとえば、フランスで MPN ID を使用していて、フランスと signed MPA の CSP 間接リセラープログラムに登録している場合、 [Csp ヨーロッパリージョン](regional-authorization-overview.md#europe-region-and-market) 全体に準拠しており、他の欧州国では MPA に署名する必要がありません。 CSP の欧州リージョン全体で間接プロバイダーを使用できます。 

### <a name="i-am-a-direct-bill-partner-in-france-and-also-transact-with-an-indirect-provider-in-italy-do-i-need-to-sign-two-mpas--one-as-direct-bill-and-one-as-indirect-reseller-"></a>フランスの直接請求パートナーであり、イタリアで間接プロバイダーを使用している場合は、2 MPAs として署名する必要があります。1つはダイレクト請求で、もう1つは間接リセラーとして使用することですか?

**回答**  CSP リージョンで既に直接請求書を使用している場合は、CSP 間接リセラー MPA に署名する必要はありません。 ただし、MPN アカウントがパートナーセンターにあり、間接プロバイダーと共有されている MPN ID がパートナーセンターにあることを確認する必要があります。 

### <a name="do-we-have-to-keep-signing-the-indirect-reseller-mpa-agreement-every-year"></a>常に間接リセラーの MPA 契約に署名する必要がありますか。


**回答** 毎年署名する必要はありません。 間接プロバイダーを使用するには、コンプライアンスを維持するために、CSP リージョンごとに少なくとも1つの MPA が署名されている必要があります。


### <a name="if-the-subscriptions-are-not-changing-does-it-matter-if-resellers-sign-the-agreement"></a>サブスクリプションが変更されていない場合、リセラーが契約に署名しても問題はありませんか。 

**回答** 間接リセラーが MPA に署名していない場合は、間接プロバイダーとのやり取りがブロックされます。 Microsoft のインセンティブは影響を受けます。 

### <a name="will-i-be-off-boarded-from-partner-center-if-i-dont-sign-the-mpa-what-are-the-exact-risks-if-resellers-dont-sign-the-mpa"></a>MPA に署名していない場合、パートナーセンターからオンボードを受けることはありませんか。 リセラーが MPA に署名していない場合、どのようなリスクがありますか。

**回答** 間接リセラーは間接プロバイダー (ディストリビューター) とのやり取りをブロックされます。 近日中に間接リセラーをオフボードする計画はありません。 

### <a name="my-reseller-says-that-the-mpa-was-already-signed-by-an-employee-who-left-the-company-do-they-need-to-sign-again"></a>リセラーは、会社を退職した従業員によって既に署名されていることが示されています。 もう一度サインインする必要がありますか。

**回答** 違います

### <a name="my-reseller-recently-changed-company-name-but-signed-the-mpa-already-do-they-need-to-sign-again"></a>最近、リセラーは会社名を変更しましたが、既にサインインしています。 もう一度サインインする必要がありますか。

### <a name="do-we-have-to-communicate-with-our-resellers-about-signing-the-mpa-or-microsoft-will-work-with-resellers-directly-to-sign--mpa"></a>MPA に署名するためにリセラーと通信する必要がありますか。または、Microsoft が再販業者と直接連携して、MPA に署名しますか?
 
**回答** 間接的なプロバイダーとして、すべての間接リセラーと明確に通信し、MPA に署名しないことによる悪影響を認識することはお客様の責任です。 また、MPA に署名する際に、適切なサポートとガイダンスを提供する必要もあります。 お客様によって solvable されない重大な問題については、このパートナーセンターサポートトピックを使用して Microsoft にリダイレクトする必要があります。

- パートナーセンターのメニューで [ **サポート** ] を選択し、[ **全般** > アカウントの確認 >] を選択して、ヘルプが必要な手順を選択します。

### <a name="my-reseller-believes-they-are-already-enrolled-as-an-indirect-reseller-because-they-have-been-able-to-resell-cloud-subscriptions-through-us-however-they-have-no-option-to-sign-the-mpa-why-is-that"></a>リセラーは、クラウドサブスクリプションを購入することができたので、既に間接リセラーとして登録されていると考えていますが、MPA に署名するオプションはありません。 なぜですか?

**回答** リセラーがパートナーセンターにサインインできても、MPA に署名するオプションや、CSP 間接リセラープログラムに登録されているかどうかを確認するオプションが表示されない場合、最も可能性が高いのは、MPN プログラムに登録されていても、間接リセラーとして CSP に登録されていないことです。 以前は、CSP プログラムに正式に登録されていない場合でも、リセラーは間接プロバイダーを使用できるようになりました。

これで、正式な CSP 間接リセラーになるようになりました。
これは、CSP で拘束力している "Advisor" パートナーにも当てはまります。 間接リセラーが間接プロバイダーとのやり取りを行うには、Microsoft は、各 CSP のビジネスリージョンで、アクティブな CSP 間接リセラーアカウントを必要とします。

### <a name="how-do-i-know-if-i-have-right-role-in-partner-center-to-sign-mpa"></a>MPA に署名するための適切なロールがパートナーセンターにあるかどうかを操作方法確認してください。

**回答** グローバル管理者のみが、MPA に署名できます。 パートナーセンターの役割を確認するには、次のようにします。

1. パートナーセンターで CSP 間接リセラーアカウントにサインインし、まず CSP 間接リセラーアカウントに登録されていることを確認します。 

2. [ **ユーザーの管理** ] にアクセスし、サインインしたアカウントに "グローバル管理者" ロールが割り当てられていることを確認します。 そうでない場合は、MPA に署名するために使用することはできません。 [ユーザー管理] ページの検索ボックスで全体管理者を検索し、それらの管理者に連絡して、MPA に署名します。

:::image type="content" source="images/user-management-clean.png" alt-text="パートナーセンターでロールを識別する":::

CSP の管理エージェントまたは販売代理店の場合は、[契約] ページにアクセスして、Microsoft パートナー契約が署名済みの詳細情報と共に表示されることを確認できます。
