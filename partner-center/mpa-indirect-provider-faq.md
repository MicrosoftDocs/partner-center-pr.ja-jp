**適切なロール**: グローバル管理者|管理エージェント |セールス エージェント

次の質問と回答は、間接リセラーが MPA を受け入れる方法と理由を理解するために役立つ潜在的な問題と解決策に対処します。 

この情報は、インダイレクト プロバイダー、直接請求パートナー、および間接リセラークラウド ソリューション プロバイダープログラムで役立ちます。

## <a name="questions-on-mpa-program-and-policy"></a>MPA プログラムとポリシーに関する質問

#### <a name="what-are-the-legal-implications-if-the-indirect-reseller-does-not-accept-the-mpa"></a>間接リセラーが MPA を受け入れない場合の法的な影響は何ですか?
 
**回答** (間接リセラー) が MPA に同意しない限り、CSP プログラムで間接プロバイダー (ディストリビューター) と取引を行う必要があります。 ビジネスを行う CSP リージョンごとに [MPA](regional-authorization-overview.md) を受け入れる必要があります。 間接リセラーが、その CSP リージョン内の間接プロバイダーと CSP トランザクションを実行するために準拠するには、CSP リージョンごとに少なくとも 1 つの MPA 署名が必要です。 

#### <a name="is-mpa-available-in-local-languages"></a>MPA はローカル言語で使用できますか?
 
**回答** いいえ。 英語でのみ使用できます。 

#### <a name="my-reseller-believes-they-are-already-enrolled-as-an-indirect-reseller-because-they-have-been-able-to-resell-cloud-subscriptions-through-us-however-they-have-no-option-to-accept-the-mpa-why-is-that"></a>リセラーは、既に間接リセラーとして登録されていると考えられます。これは、顧客がクラウド サブスクリプションを販売できたためですが、MPA を受け入れるオプションはありません。 なぜでしょうか。

**回答** リセラーが パートナー センター にサインインできるが、MPA を受け入れるオプションや、MPA が CSP Indirect Reseller プログラムに登録されているのを確認するオプションが表示されない場合は、MPN プログラムにのみ登録されているが、間接リセラーとして CSP には登録していない可能性が高い。 以前は、CSP プログラムに正式に登録していなくても、リセラーが間接プロバイダーを通じて取引を行うのを許可しました。

次は、公式の CSP 間接リセラーになります。
これは、CSP で取引を行っている "Advisor" パートナーにも適用されます。 間接リセラーが間接プロバイダー CSP Indirect Reseller処理するには、ビジネスのすべての CSP リージョンでアクティブなサービス アカウントが必要です。

#### <a name="i-work-with-multiple-indirect-providers-distributors-in-various-countries-do-i-need-to-accept-the-mpa-in-each-csp-country"></a>さまざまな国で複数の間接プロバイダー (ディストリビューター) と仕事をしています。 CSP の国ごとに MPA を受け入れる必要がありますか?


**回答** MPN ID に対して、CSP リージョンごとに少なくとも 1 つの MPA 署名が必要です。 たとえば、フランスに MPN ID を持ち、フランスの CSP Indirect Reseller プログラムに登録し、MPA を受け入れる場合 [、CSP](regional-authorization-overview.md#europe-region-and-market) ヨーロッパ リージョン全体に準拠しており、他のヨーロッパの国で MPA を受け入れる必要がない場合です。 CSP ヨーロッパ リージョン全体の間接プロバイダーと取引できます。 

#### <a name="i-am-a-direct-bill-partner-in-france-and-also-transact-with-an-indirect-provider-in-italy-do-i-need-to-accept-two-mpas-one-as-direct-bill-and-one-as-indirect-reseller-"></a>私はフランスの直接請求パートナーであり、イタリアの間接プロバイダーと取引しています。2 つの MPA (1 つは直接請求、もう 1 つは間接リセラー) を受け入れる必要がありますか?

**回答**  CSP リージョンで直接請求 MPA を既に受け入れる場合は、CSP 間接リセラー MPA を受け入れる必要があります。 ただし、MPN アカウントが パートナー センター 内に、間接プロバイダーと共有されている MPN ID が パートナー センター。 

#### <a name="do-we-have-to-keep-accepting-the-indirect-reseller-mpa-agreement-every-year"></a>間接リセラー MPA 契約に毎年同意し続ける必要がありますか?


**回答** 毎年受け入れる必要はない。 間接プロバイダーと取引するには、コンプライアンスを維持するために、CSP リージョンごとに少なくとも 1 つの MPA を受け入れる必要があります。


#### <a name="if-the-subscriptions-are-not-changing-does-it-matter-if-resellers-accept-the-agreement"></a>サブスクリプションが変更されていない場合、リセラーが契約に同意する場合は重要ですか? 

**回答** 間接リセラーが MPA を受け入れなかった場合、間接プロバイダー との取引はブロックされます。 Microsoft インセンティブは影響を受け取る可能性があります。

**回答** 間接リセラーは、間接プロバイダー (ディストリビューター) との取引をブロックされます。 近い将来、間接リセラーをオフボードする予定はありません。

## <a name="questions-about-csp-regional-authorization"></a>CSP のリージョン承認に関する質問

#### <a name="what-is-a-csp-region"></a>CSP リージョンとは

**回答** CSP リージョンは、CSP ビジネスを実行できる 1 つ以上の国で構成されます。 ビジネスには、間接プロバイダーの操作、顧客との接続、取引、インセンティブの発生などです。 CSP リージョンでビジネスを行うには、少なくとも 1 つの CSP アカウント (Office 365 仕事用アカウント) が必要です。 [詳細については、CSP リージョンに関するページを参照してください](regional-authorization-overview.md)。

顧客のCSP Indirect Reseller、同じリージョン内の間接プロバイダーと提携する必要があります。また、同じリージョン内の顧客にのみ販売できます。

#### <a name="why-cant-i-associate-my-mpn-id-to-my-tenant"></a>MPN ID をテナントに関連付けできない理由

**回答**  MPN ID をテナントに関連付けできず、エラーが表示される場合は、MPN ID とテナントが同じ国にいないことを意味します。 関連付けを機能するには、両方とも同じ国にいる必要があります。 この要件は、Microsoft のインセンティブが国と MPN ID レベルごとに支払われるためです。 たとえば、フランスに CSP アカウントがある場合、有料のインセンティブを受け取るには、フランスでも MPN の場所 (MPN ID) が必要です。 英国の MPN ID。 は、同じ CSP リージョンにある場合でも機能しません。 

これを解決するには、CSP の国と同じ国に MPN の場所を作成します。 MPN の場所を管理 [する方法の詳細については、を参照してください](manage-locations.md)。

#### <a name="i-have-a-csp-indirect-reseller-account-in-france-can-i-partner-with-indirect-provider-in-germany"></a>フランスにCSP Indirect Resellerアカウントがあります。 ドイツの間接プロバイダーと提携できますか?

**回答** はい。 CSP リージョンCSP Indirect Resellerアカウントがアクティブな場合は、同じアカウントを使用して、リージョン全体でビジネスを行います。 [詳細については、CSP リージョンに関するページを参照してください](regional-authorization-overview.md)。

#### <a name="i-am-an-indirect-reseller-in-the-us-can-i-partner-with-an-indirect-provider-in-uk"></a>私は米国の間接リセラーです。 英国の間接プロバイダーと提携できますか?

**回答** いいえ。 CSP は地域プログラムです。 ビジネスを行うには、間接リセラーと間接プロバイダーの両方の CSP アカウントが同じ CSP リージョンにある必要があります。 これを解決するには、ヨーロッパ リージョンに新しい CSP アカウントを作成し、CSP アカウントの国と同じ国に MPN の場所を作成してから、間接プロバイダーからのリレーションシップ要求に同意する必要があります。 たとえば、フランスで CSP Indirect Reseller アカウントを作成し、フランスに MPN の場所を作成した後、フランスまたは CSP ヨーロッパ リージョンの他の国にある間接プロバイダーからの招待を受け入れすることができます。

#### <a name="can-i-have-more-than-one-mpn-and-csp-accounts-in-partner-center"></a>複数の MPN アカウントと CSP アカウントを作成パートナー センター。

**回答** 会社が行っているビジネスの CSP リージョンに応じて、1 つの MPN アカウント (サインイン) と 1 つ以上の CSP アカウント (サインイン) のみを使用できます。 たとえば、米国 (HQ) に MPN アカウントを持ち、米国 (子会社)、英国 (子会社)、オーストラリア (子会社) に場所を指定できます。
MPN アカウントと同じサインイン資格情報を持つ 1 つの CSP アカウント (Office 365 仕事用アカウント) を米国内に持つ場合があります。 英国で 2 つ目の CSP アカウントを持つ場合があります。 ヨーロッパ リージョンで CSP ビジネスを管理するための異なるサインイン資格情報を持ち、オーストラリアでは 3 番目の資格情報を持ち、オーストラリアでの CSP ビジネスを管理するためのサインイン資格情報は他の 2 つとは異なります。

これらの子会社でビジネスを行い、インセンティブを獲得するには、3 つの CSP アカウントすべてが MPN ID の場所アカウントに関連付けられている必要があります。 MPN コンピテンシーと特典は、MPN アカウントでのみ管理できます。

#### <a name="i-am-a-csp-indirect-reseller-in-us-and-want-to-expand-my-business-to-europe-what-do-i-need-to-do-in-partner-center"></a>私は米国CSP Indirect Resellerで、ヨーロッパへのビジネスの拡大を望んでいます。 この手順で行う必要パートナー センター?

**回答** ビジネスを行う国に MPN の場所が必要です。また、同じ国の CSP Indirect Reseller プログラムに登録する必要があります。 新しい MPN の場所と [CSP 登録の作成の詳細については、を参照してください](manage-locations.md#prerequisites-in-order-to-add-a-new-account-for-a-csp-business)。

## <a name="questions-about-working-in-partner-center"></a>データの操作に関するパートナー センター

#### <a name="i-dont-know-who-the-partner-center-admin-is-who-can-accept-mpa"></a>MPA を受け入れパートナー センター管理者が誰かわかりません。

**回答** 会社のグローバル管理者は、MPA を受け入れる必要があります。 会社の誰がグローバル管理者のロールを割り当てられているのか確認するには、[ユーザー管理] に **移動します。** [検索] ボックスに、「グローバル管理者」と入力します。グローバル管理者ロールが割り当てられているユーザーの名前がプルされます。 これらのユーザーは誰でも MPA を受け入れできます。 会社にグローバル管理者がいない場合は、「会社にグローバル管理者が必要な場合 [の操作」を参照してください](no-mpn-admin.md)。

:::image type="content" source="images/user-management-clean.png" alt-text="グローバル管理者を検索する方法。":::


#### <a name="my-reseller-says-that-in-partner-center-they-dont-see-an-option-to-accept-the-mpa-how-can-i-best-support-them"></a>リセラーは、このパートナー センター MPA を受け入れるオプションが表示できないと言っています。 それらを最適にサポートするにはどうすれば良いですか?

**回答**  リセラーがサインインできるパートナー センター MPA を受け入れるオプションが表示しない場合:

1. 左側の [CSP] タブに [概要] ページが表示されますか? 

:::image type="content" source="images/csp.png" alt-text="[オプション] メニューで CSP をパートナー センター場所。":::

[パートナー センター] メニューの [CSP] に [概要] が表示されない場合は、CSP プログラムにまだ登録していないという意味です。 リセラーは、「CSP プログラムで間接プロバイダーを使用する」の説明にCSP Indirect Resellerとして登録するプロセス [を開始する必要があります](indirect-reseller-tasks-in-partner-center.md)。 登録が完了すると、MPA に同意できます。

2. [概要] が表示された場合は、[設定] [アカウント設定] [法的パートナーまたはリセラー] タブに移動し、[  ->    ->    ->  プログラム情報] まで下にスクロールします。  間接リセラーの状態の横に表示される状態は何ですか?


status が Active 以外の場合 **は**、法的ビジネス プロファイルの検証がまだ進行中か拒否済みです。 そのページで、リセラーは検証の状態を確認し、[詳細] をクリックして修正する必要があります。これにより、リセラーは検証を続行するために必要なドキュメントを提供できます。

:::image type="content" source="images/active-status.png" alt-text="状態はアクティブです。":::

リセラーが法的ビジネス プロファイル チェックに合格してサポートを必要とする場合は、次のオプションを選択してチケットを開く必要があります。

- パートナー センター メニューで [サポート] を選択し、[全般] > [アカウントの検証>] を選択し、ヘルプが必要な手順を選択します。


#### <a name="how-do-i-know-if-i-have-right-role-in-partner-center-to-accept-mpa"></a>操作方法 MPA を受け入れるのに適切パートナー センターロールを持っているかどうかはわかりませんか?

**回答** MPA を受け入れできるのはグローバル管理者のみです。 ユーザーのロールを確認するには、次パートナー センターします。

1. パートナー センター アカウント CSP Indirect Resellerにサインインし、パートナー センターアカウントに登録CSP Indirect Resellerします。 

2. [ユーザー管理 **] に** 移動し、サインインしたアカウントに "グローバル管理者" ロールが割り当てられているか確認します。 そうではない場合は、MPA を受け入れるのに使用できません。 [ユーザー管理] ページの検索ボックスでグローバル管理者を検索し、MPA に同意する連絡を取ります。

:::image type="content" source="images/user-management-clean.png" alt-text="ユーザーのロールを特定パートナー センター。":::


#### <a name="how-do-i-check-if-my-company-has-accepted-the-mpa"></a>操作方法が MPA に同意したのか確認してください。

**回答** アカウントを使用パートナー センターサインインCSP Indirect Resellerします。
CSP で管理エージェントまたは営業エージェントを使用している場合は、[契約] ページに移動して、サブスクリプションが表示Microsoft Partner Agreement確認できます。 

:::image type="content" source="images/agreement-new.png" alt-text="[アカウント設定] で契約を見つける。":::

:::image type="content" source="images/banner-boxed.png" alt-text="緑色のバナー。":::

**回答** 自分のアカウントパートナー センターを使用してCSP Indirect Resellerサインインします。 CSP の概要 ページに移動します。  [同意して続行] ボタンが表示されない場合に、MPA が受け入れられることを示す緑色のバナーがない場合は、MPA を受け入れる適切なロールがないことを意味します。 MPA を受け入れるには、グローバル管理者に連絡する必要があります。
