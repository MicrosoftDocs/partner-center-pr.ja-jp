# <a name="technical-support-for-on-premises-or-cloud-products"></a>オンプレミスまたはクラウド製品のテクニカルサポート

**適切なロール**:
- MPN に関連付けられたグローバル管理者または MPN パートナー管理者が、テクニカルサポートの特典を有効にします。
- アクティブ化後にアクセス ID と契約 ID を表示するための、MPN に関連付けられたユーザーアカウント。
- 管理エージェントを使用して、顧客の代わりにクラウド製品のサポートを作成します。
- お客様またはお客様の組織に代わって、オンプレミスのサポートリクエストを作成するための任意のアカウント AAD または MSA。

## <a name="overview"></a>概要

[ [技術的なメリット](https://partner.microsoft.com/dashboard/mpn/membership/benefits/technical) ] ページには、2つの異なる利点に関する情報が表示されます。

1. **テクニカル製品サポートインシデント-以下をご覧ください**
2. 技術のプリセールスおよび展開サービス (TPD) – [詳細情報](technical-benefits.md)

この記事は、Microsoft Partner Network (MPN) のメンバーを対象としています。これらのメンバーは、技術的な特典をアクティブ化する方法、アクセス ID と契約 ID を表示する方法、サポートされている製品とバージョンを知る方法、MPN パートナー特典

### <a name="steps"></a>手順

1. 技術的な特典を有効にします (グローバル管理者または MPN パートナー管理者が必要です)。 アクティブ化した後、すべての MPN ユーザーアカウントがアクセス ID と契約 ID を表示して、特典を自分のアカウントに関連付けることができます。

2. テクニカルサポートの権利とサポートされている製品を確認して、お客様の製品とバージョンに MPN の特典をご利用いただけないかどうかをご確認ください

3. マイクロソフト製品サポート (最近のオンプレミス製品向け): オンプレミス製品サポートに対するビジネスのサポートを使用する

4. 署名クラウドサポート (クラウド製品用): 管理エージェントロールを持つユーザーは、 **顧客とサービス管理** を選択して要求を作成します。 各手順の詳細については、次を参照してください。

## <a name="view-and-activate-your-technical-support-benefits"></a>テクニカルサポート特典の表示とアクティブ化

パートナーセンターで技術的な特典を表示またはアクティブ化する方法:

1. MPN に関連付けられているアカウントを使用して、パートナー センターの[ダッシュボード](https://partner.microsoft.com/dashboard)にサインインします。

   > [!NOTE]
   > アカウントが CSP にのみ関連付けられていて、MPN に関連付けられていない場合は、"この操作は許可されていません" というエラーメッセージが表示されます。 MPN に関連付けられているアカウントが必要です。

2. 左側のナビゲーション メニューで **[特典]** を展開し、 **[技術関連の特典]** を選択します。 [技術的なメリット] ページには、組織のアクティブなゴールド、シルバー、または Microsoft Action Pack メンバーシッププランが表示されます。

   :::image type="content" source="images/mpn-benefits-technical-support/technical-benefits-1.png" lightbox="images/mpn-benefits-technical-support/technical-benefits-1.png" alt-text="技術的なメリットを活用して特典を拡大" :::

   > [!NOTE]
   > アクティブなメンバーシッププランがない場合は、"まだ特典がないようです" というメッセージが表示されます。 会社がアクティブなメンバーシップオファーを持っていても、表示されない場合は、間違ったアカウントを使用してサインインしています。 アカウントを確認するには、ページの右上にある **アカウント** アイコンを選択し、[ **別のアカウントでサインイン** する (必要な場合)] を選択します。

3. マイクロソフト製品サポートまたは署名製品サポートを使用するには、特典を有効にする必要があります。 特典を有効にするには、[ **アクティブ化** ] ボタンを選択します。 ライセンス認証に使用するユーザーアカウントには、全体管理者ロールまたは MPN パートナー管理者ロールが必要です。

   :::image type="content" source="images/mpn-benefits-technical-support/technical-benefits-2.png" lightbox="images/mpn-benefits-technical-support/technical-benefits-2.png" alt-text="技術的な利点選択アクティブ化" :::

4. アクティブ化が成功すると、アクセス ID と契約 ID がすべてのユーザーに表示されます。 アクセス ID と契約 ID が表示されている場合は、Microsoft 製品サポートと署名製品サポート (Gold およびシルバーメンバー) を使用できます。

   [アクティブ化] ボタンがグレーで表示されている場合、アカウントにはグローバル管理者または MPN Partner Admin セキュリティロールがありません。 グローバル管理者または MPN パートナー管理者の  [ユーザー管理](https://partner.microsoft.com/dashboard/account/v3/usermanagement#users) とフィルターを使用して、特典を有効にするユーザーを見つけます。

   :::image type="content" source="images/mpn-benefits-technical-support/technical-benefits-3.png" lightbox="images/mpn-benefits-technical-support/technical-benefits-3.png" alt-text="アクティブ化された技術的な特典アクセス ID と契約 ID を表示" :::

   > [!NOTE]
   > アカウントごとにマイクロソフト製品サポートまたは署名製品サポートを初めて使用するときは、アクセス ID と契約 ID が必要になります。 お客様のアカウントに特典が関連付けられたら、アクセス ID と契約 ID をもう一度使用する必要はありません。  

## <a name="technical-support-entitlement-and-supported-products"></a>テクニカルサポートの権利とサポートされる製品

テクニカルサポートは、特定の問題、エラーメッセージ、または想定どおりに動作しない機能に関するトラブルシューティングを提供します。 MPN 製品サポートおよび署名クラウドサポートは、特定の Microsoft 製品およびバージョンに対して使用できます (一部の製品とバージョンがサポートされているわけではありません)。

> [!NOTE]
> 使用するには、テクニカルサポートの特典をアクティブ化する必要があります。

Microsoft Partner Network のメンバーシップの一部として、パートナーは次のテクニカルサポートの対象になります。

|  テクニカル サポート |  Action Pack |  シルバーコンピテンシー | Gold コンピテンシー |
|:------------------:|:------------:|:------------------:|:---------------:|
| マイクロソフト製品サポート<br>最新のオンプレミス用 <br>製品のみ | 10件のインシデント | 15インシデント | 20インシデント |
| 署名クラウドのサポート<br>クラウド製品のみ | | クラウドコンピテンシー:<br>無制限<br><br>その他のコンピテンシー:<br>5インシデント | クラウドコンピテンシー:<br>無制限<br>          <br>その他のコンピテンシー:<br>10件のインシデント   | 

クラウドコンピテンシーには、セキュリティコンピテンシー、クラウド Business Applications、クラウドプラットフォーム、クラウドの生産性、エンタープライズモビリティ管理、小規模および思い浮かべのクラウドソリューションなどがあります。

## <a name="microsoft-product-support-on-premises"></a>Microsoft 製品サポート (オンプレミス)

最新のオンプレミス製品のテクニカルサポートについては、 **Microsoft 製品サポート** をご利用いただけます。 最新のオンプレミス製品には、現在の製品 (N) と以前の製品バージョン (N-1) のみが含まれます。 以前の製品バージョンは、 [メインストリームサポート](/lifecycle/policies/fixed)内にある場合にのみサポートされます。 [N-1](https://aka.ms/n-1) のサポートポリシーに基づきます。 サポートされていない製品とバージョンについては、関連する有料サポートオプションを使用するか、パートナーサポートプランを購入してください。

特典を初めて使用するときは、アクセス ID と契約 ID が必要になります。 特典は、有効期限が切れるまで、またはサポートインシデントが残っていない限り、ユーザーアカウントに関連付けられます。

すべてのユーザーアカウント (AAD または MSA) は、アクセス ID と契約 ID を持っている場合、オンプレミス製品のテクニカルサポートを利用できます。

オンプレミス製品のサポートリクエストを作成するには、[ [ビジネス向けサポート](https://support.serviceshub.microsoft.com/supportforbusiness/create) ] を選択します。

### <a name="how-many-partner-support-incidents-are-remaining"></a>パートナーサポートインシデントが残っている数

この情報は、ビジネス向けのオンプレミスサポートポータルからのみ取得できます。 残念ながら、クラウドポータル (Azure、LCS など) では、この情報は表示されません。

オンプレミスのサポートインシデントを作成するには、ビジネス向けのサポートを使用し、プロセスを開始する必要があります。 プロセスを完了しないため、サポート要求は作成されません。 必要な情報は、手順 2. で表示されます。

1. [ビジネス向けサポート](https://support.serviceshub.microsoft.com/supportforbusiness/create)を開き、windows 10 を選択してダミー情報を入力します

2. サポート契約が既にアカウントに関連付けられている場合は、サポートインシデントが表示されます。 オンプレミスのサポートについては、[開始日]、[終了日]、および [残りのインシデント] を含め、パートナーサポート契約が表示されていることを確認する必要があります (無効になっていない場合)。

サポート契約を関連付ける必要がある場合は、[購入または追加のサポートプランを追加する必要があります] を展開し、[ **契約の追加** ] ボタンを選択してアクセス ID と契約 ID を入力します。 [**使用できないサポートプランを表示しますか?] を展開します。** パートナーの署名情報を表示するセクション。 ここに表示されるのは、契約の有効期限が切れた場合、インシデントが残っていない場合、または古いオンプレミス製品を選択した場合のみです (サポート製品については、 [aka.ms/n-1](https://aka.ms/n-1) を参照してください)。

詳細については、「 [パートナーサポートインシデントが残っている数](https://support.microsoft.com/topic/how-many-partner-support-incidents-are-remaining-e8220db5-14f6-93db-4b4a-35766e11b3c3)」を参照してください。

## <a name="signature-cloud-support-cloud-only"></a>署名クラウドのサポート (クラウドのみ)

Gold またはシルバーコンピテンシーパートナーは、クラウド製品のテクニカルサポートのために、 **署名クラウドサポート** をご利用いただけます。 Action Pack パートナーは、コンピテンシーを獲得しない限り、署名クラウドサポートの対象にはなりません。  

Azure、Dynamics 365、およびその他のクラウド製品の特典を初めて使用するときは、アクセス ID と契約 ID が必要になります (例外 Microsoft 365)。 特典は、有効期限が切れるまで、またはサポートインシデントが残っていない限り、ユーザーアカウントに関連付けられます。

> [!NOTE]  
> Azure portal を使用する場合。 サポート契約が既にアカウントに関連付けられている場合は、"アクセス ID は既に関連付けられています..." というエラーメッセージが表示されます。アクセス ID と契約 ID を追加する場合。 以下を参照

Azure 要求を作成する CSP パートナーは、サポート契約を追加せずに "Cloud Solution Provider" サポートプランを使用できます。

**管理エージェント** ロールを持つユーザーは、左側のナビゲーションメニューで [CSP] の下にある [顧客](https://partner.microsoft.com/commerce/customers/list)を選択し、[**サービス管理**] を選択して、[**新しい要求**] を選択できます。  関連するワークロードを選択すると、ワークロード固有のポータルが開き、サポート契約を追加して要求を作成します。

### <a name="azure-portal-error-access-id-is-already-associated-"></a>Azure portal エラー "アクセス ID は既に関連付けられています..."

Azure portal を使用する場合は、[ヘルプとサポート]、[サポートプラン] の順に選択し、[サポートの特典をリンクする] を選択し、アクセス ID と契約 ID を入力して、署名クラウドサポートを使用します。

> [!NOTE]
> Microsoft Action Pack お持ちの場合は、クラウドサポートを利用できません。これには、Microsoft 製品サポート (オンプレミス) の署名が含まれていないためです。

エラーメッセージが表示されることがあり `This Access ID is already associated to an account. Please contact your Technical Account Manager to obtain a new Access ID` ます。 このエラーメッセージは無効であるため、無視する必要があります。 エラーを無効にし、役に立つ診断エラーメッセージに置き換えるように要求しました。

`Access ID is already associated …`Azure portal を使用するときに、サポート契約を1つのアカウントに複数回関連付けると、上記のエラーが表示されます。 以下のシナリオを参照してください。

1. Gold またはシルバーの技術的な特典があり、既にオンプレミスサポートのコントラクトを追加している場合は、Azure portal を使用し、同じアカウントにもう一度契約を追加します。 サポート契約をアカウントに関連付ける必要があるのは1回だけであるため、Azure のサポート特典をリンクする必要はありません。 コントラクトが既にアカウントに関連付けられている場合にリンクサポート特典を使用すると、エラーが表示され、無視する必要があります。 + New Support request を使用し、通常どおりサポートインシデントを作成して、サポートプランを表示します。

2. サポート契約の有効期限が切れている場合、または署名10パック (Gold ハイブリッドコンピテンシー) または署名5パック (シルバーハイブリッドコンピテンシー) のインシデントが残っていない場合、サポート契約は無効であるため、非表示になります。 サポート特典を再びリンクすると、コントラクトが既に関連付けられているため、エラーが表示されます。 新しいサポートプランを購入して、Azure サポートインシデントを取得することができます。

パートナーの有料サポートオファリングについては、以下のパートナーサポートプランを参照してください。

## <a name="partner-support-plans"></a>パートナーサポートプラン

マイクロソフトから有償サポートを受けることができます。これには、テクニカルサポートと、その他のさまざまなサービスオプションが含まれます。

パートナーの [サポートプランを比較](https://partner.microsoft.com/support/partnersupport) するか、パートナー固有の計画について以下を参照してください。

- [パートナー向けの Premier サポート](https://partner.microsoft.com/support/microsoft-services-premier-support) -Microsoft のテクニカルサポートのトップレベル
- [パートナー向けの高度なサポート](https://partner.microsoft.com/support/advanced-cloud-support) -クラウド製品のみ
